# 文本分割策略工厂模块

## Summary
在 `module/ai/shared/textsplit` 下新增通用文本分割模块，采用“输入类型正常化 + 分割模式执行”的策略工厂结构，替换当前 `AiKnowledgeChunkServiceImpl` 内部的单一 `splitText` 实现。模块只负责分割，不涉及上传、存储、语义分析。

## Key Changes
### 接口与类图
```text
TextSplitStrategyFactory
├─ resolve(inputType, splitMode) -> TextSplitStrategy

TextSplitStrategy
└─ split(TextSplitSource source, TextSplitOptions options) -> List<String>

TextSourceNormalizer
└─ normalize(TextSplitSource source) -> String

TextSplitModeStrategy
└─ split(String text, TextSplitOptions options) -> List<String>
```

### 核心定义
```java
public interface TextSplitStrategy {
    List<String> split(TextSplitSource source, TextSplitOptions options);
}

public interface TextSourceNormalizer {
    boolean supports(TextInputType inputType);
    String normalize(TextSplitSource source, TextSplitOptions options);
}

public interface TextSplitModeStrategy {
    boolean supports(SplitMode splitMode);
    List<String> split(String text, TextSplitOptions options);
}

public interface TextSplitStrategyFactory {
    TextSplitStrategy resolve(TextInputType inputType, SplitMode splitMode);
}
```

### 示例实现
- 输入类型策略
  - `PlainTextNormalizer`
  - `PdfTextNormalizer`
- 分割模式策略
  - `FixedLengthSplitModeStrategy`
  - `ParagraphSplitModeStrategy`

### 扩展位
- 输入类型：`MarkdownTextNormalizer`、`HtmlTextNormalizer`、`DocxTextNormalizer`
- 分割模式：`SentenceSplitModeStrategy`、`RegexSplitModeStrategy`

### 业务接入
- `AiKnowledgeChunkServiceImpl` 仅保留分块编排和向量化。
- 原 `splitText` 删除，改为通过工厂获取策略后执行分割。
- 现有 `ai.rag.chunkSize / chunkOverlap` 作为固定长度分割默认参数复用。

## Test Plan
- 纯文本 + 固定长度分割
- 纯文本 + 段落分割
- PDF + 固定长度分割
- PDF + 段落分割
- Markdown / HTML / DOCX 路由到对应 normalizer
- 未知输入类型或未知分割模式应快速失败
- 回归验证 `AiKnowledgeChunkServiceImpl` 输出块数量与内容稳定

## Assumptions
- Java 17，避免依赖更激进的语言特性。
- PDF/DOCX 解析优先复用项目现有的 Tika 相关依赖。
- 工厂只负责选择，不写 `switch` 业务分支。
- 新增输入类型或分割模式只加策略类，不改工厂核心逻辑。
