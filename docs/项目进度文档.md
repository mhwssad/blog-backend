# 项目进度文档

本文档用于记录当前仓库的真实落地进度，面向项目内部协作、排期和后续验收使用。

## 1. 统计口径

- 统计时间：截至 2026-04-01
- 统计范围：`src/main/java/com/cybzacg/blogbackend` 与 `docs`、`src/test`
- 当前定位：单体博客后端，核心业务模块为 `auth`、`article`、`content`、`file`

## 2. 当前整体进度概览

### 2.1 已落地主线

- 认证与鉴权主线已具备可运行骨架，包含登录、注册、邮箱验证码登录、刷新令牌、退出登录。
- 后台 RBAC 主线已落地，包含用户、角色、菜单、配置、通知、日志等管理接口。
- 内容域主线已落地，包含文章、分类、标签、评论、收藏、互动、足迹等核心能力。
- 文件域主线已落地，包含用户文件上传、分片上传、秒传复用、后台文件管理。
- 已补入聊天域首版建表脚本，为单聊、群聊、全站特殊群聊以及消息回执能力预留数据库基础。
- 粉丝关注主线已落地首版用户侧接口，当前已支持关注、取关、关注列表、粉丝列表、互关判断、实时统计、特别关注和备注维护。
- 已新增数据库表设计修复文档与存量库修复脚本，开始收口实库缺失表、唯一约束、关键索引和日志表引擎不一致问题。

### 2.2 当前规模

- 业务模块数：6 个
- 控制器数量：28 个
- `ServiceImpl` 数量：67 个
- 已存在测试文件：59 个
- 已存在前端接口文档：`auth-api.md`、`content-api.md`、`file-api.md`、`chat-api.md`、`follow-api.md`

### 2.3 当前判断

- 项目已经从“只有框架骨架”进入“核心业务接口基本齐备”的阶段。
- 当前主要短板不在模块空缺，而在测试隔离覆盖范围、边界场景验证、文档持续维护和部分能力的收尾一致性。
- 聊天 / WebSocket 方向已完成 chat v1 首版实现，包含单聊 / 群聊 / 全站群聊、文本 / 文件 / 图片 / 语音消息、基础回复与 reply 快照、会话列表 / 历史消息、已读游标、群管理、后台聊天管理、基于 Redis 广播的多节点 WebSocket 实时推送，以及持久化异步媒体任务、聊天域频控/敏感词和指标埋点。
- 粉丝关注方向已完成用户侧主链路、后台治理接口、公开访客接口与新粉丝通知联动，当前剩余重点收敛到主页展示边界、缓存评估和更深层集成验证。
