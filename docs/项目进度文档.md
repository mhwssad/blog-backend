# 项目进度文档

本文档只记录当前项目进度概览，用于快速了解项目已经做到哪里、下一步从哪里继续。详细任务拆分统一维护在 `docs/tasks/README.md`。

## 1. 当前阶段

- 当前定位：以博客为核心、社区互动为扩展方向的单体后端项目。
- 当前阶段：核心业务能力已基本成型，正在进入新 PRD 能力补齐和质量收口阶段。
- 当前主线：`60% 博客 + 40% 社区`，后续逐步接入 AI、论坛、知识库 / RAG、agents 等能力。
- 当前任务入口：`docs/tasks/README.md`。

## 3. 已完成基础能力

- 认证与权限：账号密码登录、邮箱验证码登录、注册、刷新令牌、退出登录、RBAC 后台管理。
- 博客内容：文章管理、公开查询、分类、标签、评论、收藏、点赞、足迹、附件引用。
- 文件能力：上传初始化、普通上传、分片上传、秒传、用户文件列表、后台文件管理、生命周期清理。
- 聊天能力：会话、消息、群管理、WebSocket 推送、Redis 多节点广播、后台聊天治理。
- 关注关系：关注 / 取关、关注 / 粉丝列表、互关判断、公开访客查询、后台治理。
- 工程结构：Repository 层迁移已完成当前轮次，项目结构规范和代码编写规范已建立。
- 文档体系：`docs/README.md`、`docs/api文档`、`docs/需求文档`、`docs/tasks` 已形成当前文档入口。

## 4. 当前主要缺口

- 博客侧：文章多状态、审核流转、定时发布、白名单 / 登录可见、系列文章仍需按新 PRD 补齐。
- 社区侧：大厅频道、主题频道、频道申请、群入群审批、邀请链接、群分类 / 搜索仍需实现。
- AI 侧：AI 问答主链路、配置中心、额度控制、数据读取范围尚未实现。
- 治理侧：举报处理、处罚留档、超级管理员高风险审计、通知偏好和数据看板仍需补齐。
- 测试侧：模块服务级测试已有基础，但新 PRD 能力仍需要随实现补测试。

## 6. 维护规则

- 本文档只维护项目进度概览，不记录详细任务过程。
