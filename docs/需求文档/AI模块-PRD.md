## 2. 模块定位

AI 模块第一阶段定位为平台统一问答入口，为用户提供基础 AI 能力，并为后续知识库 / RAG / agents 扩展提供统一配置和治理基础。

当前项目已完成第一阶段基础问答、会话记录、渠道配置、额度控制和调用统计。第二期定位为在既有 AI 主链路上扩展知识库、RAG、agents、工具调用和 MCP 接入，使 AI 从通用问答升级为可引用平台内容、可受权限治理约束、可执行异步任务、可调用受控外部能力的智能能力中心。

## 3. 模块目标

- 提供全站统一 AI 问答入口。
- 建立独立 AI 页面和后台配置中心。
- 建立可配置、可限额、可控数据范围的 AI 接入方式。
- 为后续知识库和 agents 扩展保留统一入口。
- 建立二期知识库 / RAG 能力，支持可追溯、可权限过滤的知识增强问答。
- 建立 agents 定义与任务执行基础，支持后续扩展自动化工作流。
- 建立 AI 工具调用与 MCP 接入能力，支持模型在授权范围内调用平台内工具和外部 MCP 服务。

## 4. 目标用户

- 注册用户：使用 AI 问答能力。
- 管理员：配置模型、额度、数据范围与开关。
- 超级管理员：处理高风险配置和特殊问题。

## 5. 第一阶段范围

第一阶段 AI 能力包括：

- 全站统一 AI 助手入口
- 单独 AI 页面
- 后台 AI 配置中心
- 基于通用大模型的问答能力
- 对话记录保存
- 用户级限额控制

## 5.1 当前已完成基线

当前 AI 模块已具备：

- 用户侧 AI 会话创建、列表、详情和消息发送。
- 用户侧 AI 配额查询。
- 后台 AI 渠道配置、启用 / 停用。
- 后台 AI 会话分页、调用日志分页和调用统计。
- 用户级额度控制和调用日志留痕。
- 数据读取范围配置基础。

当前尚未完成：

- 知识源、知识条目和同步任务。
- 文档切片、向量化、向量索引和检索。
- RAG 上下文拼装、引用返回和无命中策略。
- agent 定义、任务发起、任务执行和任务通知。

## 5.2 第二期范围

第二期 AI 能力包括：

- 知识源管理。
- 知识条目同步与状态追踪。
- RAG 检索增强问答。
- RAG 引用来源返回。
- RAG 权限过滤。
- agent 定义管理。
- 用户发起 agent 任务。
- agent 任务状态查询。
- 工具定义、授权、调用执行和调用日志。
- MCP 服务配置、连接状态管理、工具发现和调用审计。
- AI / RAG / agent 统一额度和调用日志。

## 6. 核心功能需求

### 6.1 AI 问答入口

需支持：

- 全站统一入口
- 独立 AI 页面访问
- 登录用户使用

### 6.2 对话能力

第一阶段以通用大模型问答为主，不要求知识库检索或复杂上下文增强。

### 6.3 对话记录

AI 对话记录需要保存，并在后续支持分类化管理。

### 6.4 使用限制

需支持：

- 每日调用次数限制
- 上下文长度限制
- 用户级限额

### 6.5 知识库

第二期知识库需支持：

- 知识源类型：公开文章、作者公开资料、论坛帖子、后台维护知识条目。
- 知识源启用 / 停用。
- 知识条目同步状态：待同步、同步中、已同步、同步失败。
- 知识条目内容快照，避免源内容变化后无法追溯。
- 同步失败原因记录和重试入口。

### 6.6 RAG 问答

第二期 RAG 需支持：

- 根据用户问题检索可访问知识片段。
- 将命中片段拼装进模型上下文。
- 回答结果返回引用来源。
- 无命中时允许降级为普通问答或返回无命中提示。
- 调用日志记录是否启用 RAG、检索耗时、命中数量。

### 6.7 Agents

第二期 agents 需支持：

- 后台创建、编辑、启用、停用 agent。
- agent 配置项：名称、描述、提示词、可用工具、数据范围、启用状态。
- 用户发起 agent 任务。
- 任务状态：待执行、执行中、已完成、执行失败、已取消。
- 任务输入、输出、错误信息和执行耗时留存。
- agent 完成或失败后可接入通知中心。

### 6.8 工具调用

第二期 AI 工具调用需支持：

- 后台维护工具定义，包括工具名称、描述、入参 Schema、输出 Schema、所属能力域和启用状态。
- 工具可绑定到指定 agent 或 AI 场景，不允许所有工具默认全局可用。
- 工具调用前必须完成用户身份、数据范围、业务权限、频控和风险等级校验。
- 工具调用结果必须纳入模型上下文时，应进行脱敏、长度控制和敏感字段过滤。
- 工具调用过程需记录调用日志，包括调用用户、agent / 会话、工具名称、入参摘要、执行状态、耗时、错误信息和结果摘要。
- 写操作、治理操作、通知投递、外部请求等高风险工具默认需要单独授权，并预留人工确认或后台开关。

### 6.9 MCP 接入

第二期 MCP 接入需支持：

- 后台维护 MCP 服务配置，包括服务名称、连接方式、鉴权配置、启用状态、超时时间和可用工具范围。
- 支持从 MCP 服务发现工具，并将 MCP 工具纳入统一工具注册、授权、调用和审计体系。
- MCP 调用必须复用 AI 数据读取范围和 agent 工具授权，不得绕过平台权限边界。
- MCP 服务连接失败、工具不可用、调用超时或返回异常时，应明确记录失败原因，并允许 AI 主流程降级或返回可理解错误。
- MCP 鉴权信息必须按敏感配置处理，不得进入普通响应、对话记录或模型上下文。
- 后台需能查看 MCP 服务连接状态、工具清单、最近调用结果和异常统计。

## 7. 数据与隐私边界

### 7.1 第一阶段边界

- 默认禁止 AI 读取用户私聊内容
- AI 可读取数据范围由管理员配置
- 第一阶段以通用模型回答为主

### 7.2 后续扩展方向

后续逐步支持：

- 基于文章回答
- 基于聊天回答
- 基于用户资料回答
- 知识库 / RAG
- agents

### 7.3 第二期数据边界

- RAG 不得返回用户无权访问的内容引用。
- 私聊内容默认不作为知识源；如后续允许，必须单独设计授权和审计规则。
- 未发布、仅自己可见、白名单不可访问的文章不得被无权限用户检索到。
- 隐藏、删除、封禁状态的论坛内容不得进入可用检索结果。
- agent 数据读取范围必须复用 AI 数据范围配置，不允许绕过后台治理。
- 工具和 MCP 调用不得扩大用户原本可访问的数据范围。
- MCP 鉴权密钥、令牌、连接串等敏感信息不得写入 AI 对话消息、agent 输出或普通调用日志。
- 涉及写入、删除、治理、通知、外部网络请求的工具必须具备明确授权、审计和风险控制。

## 8. 后台配置中心需求

第一阶段后台最少配置项包括：

- 模型选择
- 多渠道 / API Key 配置
- 每日额度
- 用户级限额
- 可读取数据范围
- 提示词模板
- 开关控制

## 9. 管理规则

- AI 使用权限仅开放给登录用户。
- AI 数据读取范围必须可控。
- 高风险配置修改需要纳入后台治理和审计范围。
- 知识源开关、agent 工具授权、agent 数据范围调整属于高风险配置。
- MCP 服务启停、MCP 鉴权配置、工具授权范围调整属于高风险配置。
- AI 普通问答、RAG 问答和 agent 任务必须统一纳入额度消耗规则。
- 后台必须能追踪 AI 调用、RAG 命中、agent 任务执行、工具调用和 MCP 调用情况。

## 10. 非目标

第一阶段 AI 模块不重点处理：

- 知识库检索
- RAG
- agents 正式能力
- 复杂自动工作流

第二期仍不重点处理：

- 完整低代码工作流编排。
- 自动执行高风险管理动作。
- 未经授权直接执行平台写操作或外部系统写操作。
- 绕过用户权限的全量私域数据问答。
- 面向多租户的独立知识库隔离体系。

## 11. 验收标准

- 登录用户可稳定使用 AI 问答。
- 管理员可配置模型、额度、开关和数据读取范围。
- AI 问答具备基础记录保存能力。
- AI 默认不读取用户私聊数据。
- 管理员可控制知识源和 RAG 开关。
- 用户可发起 RAG 问答并看到可追溯引用。
- 用户不可通过 RAG 获取无权访问内容。
- 管理员可配置基础 agent，用户可发起 agent 任务并查询状态。
- 管理员可配置工具和 MCP 服务，并控制其可用范围。
- AI / agent 只能调用已授权工具和 MCP 工具，调用过程可追踪、可审计、可失败降级。

## 12. 待澄清事项

- AI 对话记录的分类方案
- 提示词模板的管理粒度
- RAG 无命中时默认降级策略
- agent 可用工具的首批范围
- MCP 首批接入范围、连接方式和安全策略
- 工具调用是否需要用户侧二次确认
- agent 异步任务是否需要队列化执行
- AI / RAG / agent 额度消耗倍率
