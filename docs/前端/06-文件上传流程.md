# 06 - 文件上传流程

## 1. 概述

后端文件上传模块支持三种上传模式，前端无需手动指定模式，后端会根据文件大小与 MD5 匹配情况**自动判断**采用哪种模式：

| 模式       | 值 | 说明                                  |
|----------|---|-------------------------------------|
| **秒传**   | 1 | 服务端检测到相同 MD5 的文件已存在，直接复用，无需传输文件内容   |
| **分片上传** | 2 | 文件大小超过阈值（默认 6MB），切分为多个分片逐一上传，支持断点续传 |
| **全量上传** | 3 | 文件大小未超过阈值，一次性上传整个文件                 |

### 自动判断逻辑

```
初始化任务 → 计算文件 MD5 → 提交 init 请求
  ↓
后端判断：
  1. fileMd5 命中已有文件 → uploadMode=1（秒传）
  2. fileSize > chunkSizeThreshold → uploadMode=2（分片）
  3. 否则 → uploadMode=3（全量）
```

## 2. 上传参数限制表

| 参数                       | 值                              | 说明           |
|--------------------------|--------------------------------|--------------|
| 分片大小（chunkSize）          | **5 MB** (5,242,880 bytes)     | 每个分片的最大字节数   |
| 分片阈值（chunkSizeThreshold） | **6 MB** (6,291,456 bytes)     | 超过此大小强制分片    |
| 最大文件大小（maxFileSize）      | **100 MB** (104,857,600 bytes) | 单文件上传上限      |
| 任务过期天数（taskExpireDays）   | **2 天**                        | 未完成任务自动清理    |
| MD5 校验（enableMd5Check）   | **启用**                         | 开启秒传与分片完整性校验 |
| 允许扩展名（allowedExtensions） | **不限制**                        | 为空列表时允许所有类型  |

## 4. Step 1: 初始化上传任务

### 请求

```
POST /api/user/files/upload-tasks/init
Content-Type: application/json
Authorization: Bearer <accessToken>
```

### 请求体 (FileUploadInitRequest)

```json
{
  "originalName": "photo.jpg",           // String, 必填 - 原始文件名
  "fileSize": 10485760,                  // Long, 必填 - 文件大小（字节）
  "fileMd5": "d41d8cd98f00b204e9800998ecf8427e",  // String, 可选 - 文件MD5（秒传必传）
  "mimeType": "image/jpeg",              // String, 可选 - MIME类型
  "referenceType": "article_attachment",  // String, 可选 - 引用类型（avatar/article_attachment/comment_image/temp）
  "referenceId": null,                    // Long, 可选 - 引用对象ID
  "category": "attachment",              // String, 可选 - 业务分类（avatar/attachment/comment/temp）
  "isPublic": 1,                          // Integer, 可选 - 是否公开：0-私有，1-公开
  "totalChunks": 2,                       // Integer, 可选 - 总分片数（分片上传时传）
  "chunkSize": 5242880,                   // Long, 可选 - 分片大小（分片上传时传）
  "remark": "文章附件"                    // String, 可选 - 备注
}
```

### 响应体 (FileUploadInitVO)

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": 1001,                       // Long - 上传任务ID
    "uploadId": "a1b2c3d4e5f6",           // String - 上传标识（后续接口使用）
    "uploadMode": 2,                      // Integer - 上传模式：1=秒传，2=分片，3=全量
    "quickUploadAvailable": true,         // Boolean - 是否可秒传
    "completed": false,                   // Boolean - 是否已通过秒传完成
    "totalChunks": 2,                     // Integer - 总分片数
    "chunkSize": 5242880,                 // Long - 分片大小
    "taskStatus": 0,                      // Integer - 任务状态（0=INIT）
    "fileId": null,                       // Long - 已命中的文件ID（秒传时有值）
    "fileUrl": null,                      // String - 文件访问地址（秒传时有值）
    "businessId": null                    // Long - 业务引用ID（秒传时有值）
  }
}
```

### uploadMode 值说明

| 值 | 常量             | 含义   | 前端处理                                     |
|---|----------------|------|------------------------------------------|
| 1 | QUICK_UPLOAD   | 秒传   | 检查 `completed`，若为 `true` 直接结束，否则调用秒传检测接口 |
| 2 | CHUNKED_UPLOAD | 分片上传 | 将文件切片后逐片上传，最后调用完成接口                      |
| 3 | FULL_UPLOAD    | 全量上传 | 一次性 `multipart/form-data` 上传整个文件         |

### completed=true 处理

当 `uploadMode=1` 且 `completed=true` 时，表示文件已在初始化阶段通过秒传完成，前端**无需调用任何后续接口**，可直接使用响应中的
`fileUrl` 和 `businessId`。

```javascript
if (data.uploadMode === 1) {
    if (data.completed) {
        // 秒传直接完成，无需后续操作
        return {success: true, fileUrl: data.fileUrl, businessId: data.businessId};
    }
    // 需要调用秒传检测接口确认
    if (data.quickUploadAvailable) {
        return await quickCheck(data.uploadId);
    }
}
```

## 7. Step 2c: 分片上传

当 `uploadMode=2` 时，将文件按 `chunkSize` 切分为 `totalChunks` 个分片，逐个上传。

### 请求（单个分片）

```
POST /api/user/files/upload-tasks/{uploadId}/chunks/{chunkNumber}
Content-Type: multipart/form-data
Authorization: Bearer <accessToken>
```

参数说明：

- `{uploadId}` — 初始化时返回的上传标识
- `{chunkNumber}` — 分片序号，从 **1** 开始
- `file` — 分片二进制数据（form-data 字段）
- `chunkMd5` — 可选，该分片的 MD5 值，用于服务端校验完整性

```
------Boundary
Content-Disposition: form-data; name="file"; filename="photo.jpg.part1"
Content-Type: application/octet-stream

<分片二进制数据>
------Boundary
Content-Disposition: form-data; name="chunkMd5"

e99a18c428cb38d5f260853678922e03
------Boundary--
```

### 响应 (ChunkUploadVO)

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "uploadId": "a1b2c3d4e5f6",           // String - 上传标识
    "chunkNumber": 1,                      // Integer - 当前分片序号
    "uploadedChunks": 1,                   // Integer - 已上传分片数
    "totalChunks": 3,                      // Integer - 总分片数
    "taskStatus": 1                        // Integer - 任务状态（1=UPLOADING）
  }
}
```

### 进度计算

```javascript
const progress = (uploadedChunks / totalChunks) * 100;
// 示例：uploadedChunks=2, totalChunks=3 → progress=66.67%
```

## 9. 任务状态机

### 状态表

| 值 | 常量        | 含义       |
|---|-----------|----------|
| 0 | INIT      | 初始化，等待上传 |
| 1 | UPLOADING | 上传中      |
| 2 | MERGING   | 分片合并中    |
| 3 | COMPLETED | 已完成      |
| 4 | FAILED    | 失败，可重试   |
| 5 | CANCELLED | 已取消      |

### 状态流转图

```
  INIT(0)
    │
    ├──→ UPLOADING(1) ──→ MERGING(2) ──→ COMPLETED(3)
    │        │                  │
    │        │                  ├──→ FAILED(4)
    │        │                  │
    │        ├──→ FAILED(4)     └──→ CANCELLED(5)
    │        │
    │        ├──→ CANCELLED(5)
    │        │
    │        └──→ COMPLETED(3)  ←── 全量上传直接完成
    │
    └──→ CANCELLED(5)

  FAILED(4)
    │
    └──→ UPLOADING(1)  ←── 断点续传重试
```

合法的转换路径：

- `INIT → UPLOADING` — 开始上传
- `UPLOADING → MERGING` — 分片全部上传完成，开始合并
- `UPLOADING → COMPLETED` — 全量上传直接完成
- `UPLOADING → FAILED` — 上传过程出错
- `UPLOADING → CANCELLED` — 用户取消
- `MERGING → COMPLETED` — 合并成功
- `MERGING → FAILED` — 合并失败
- `MERGING → CANCELLED` — 合并阶段取消
- `FAILED → UPLOADING` — 失败后重试（断点续传）
- `INIT → CANCELLED` — 未开始就取消

## 10. 断点续传

### 查询未完成任务

```
GET /api/user/files/upload-tasks?page=1&size=20
Authorization: Bearer <accessToken>
```

返回当前用户的上传任务列表，前端可以筛选 `taskStatus` 为 `0(INIT)`、`1(UPLOADING)`、`4(FAILED)` 的任务进行续传。

### 判断已上传分片

续传时，前端需要先调用初始化接口（使用相同的 `fileMd5`），后端会返回已有任务信息。分片上传接口每次返回 `uploadedChunks`，前端对比
`uploadedChunks` 与 `totalChunks` 即可跳过已上传的分片：

```javascript
// 续传逻辑
const initResponse = await initTask({originalName, fileSize, fileMd5, ...});
if (initResponse.uploadMode === 2) {
    // 从 uploadedChunks + 1 开始上传
    const startChunk = (initResponse.uploadedChunks || 0) + 1;
    for (let i = startChunk; i <= initResponse.totalChunks; i++) {
        await uploadChunk(uploadId, i, chunks[i - 1]);
    }
}
```

## 11. 前端实现代码示例

```javascript
/**
 * FileUploader - 文件上传器
 *
 * 封装秒传、全量上传、分片上传的完整流程。
 */
class FileUploader {
    constructor(options = {}) {
        this.baseURL = options.baseURL || '/api/user/files/upload-tasks';
        this.concurrency = options.concurrency || 3;         // 分片并发数
        this.chunkSize = options.chunkSize || 5 * 1024 * 1024; // 5MB
        this.chunkSizeThreshold = options.chunkSizeThreshold || 6 * 1024 * 1024; // 6MB
        this.maxFileSize = options.maxFileSize || 100 * 1024 * 1024; // 100MB
        this.onProgress = options.onProgress || (() => {
        });
        this.onComplete = options.onComplete || (() => {
        });
        this.onError = options.onError || (() => {
        });
    }

    /**
     * 获取认证头
     */
    _getHeaders() {
        const token = localStorage.getItem('accessToken');
        return {Authorization: `Bearer ${token}`};
    }

    /**
     * 计算文件 MD5（使用 SparkMD5 库增量计算）
     */
    async calculateMD5(file) {
        return new Promise((resolve, reject) => {
            const spark = new SparkMD5.ArrayBuffer();
            const reader = new FileReader();
            const chunkSize = 2 * 1024 * 1024; // MD5计算用2MB分块
            const chunks = Math.ceil(file.size / chunkSize);
            let currentChunk = 0;

            reader.onload = (e) => {
                spark.append(e.target.result);
                currentChunk++;
                if (currentChunk < chunks) {
                    loadNext();
                } else {
                    resolve(spark.end());
                }
            };

            reader.onerror = () => reject(reader.error);

            function loadNext() {
                const start = currentChunk * chunkSize;
                const end = Math.min(start + chunkSize, file.size);
                reader.readAsArrayBuffer(file.slice(start, end));
            }

            loadNext();
        });
    }

    /**
     * 将文件切片
     */
    sliceFile(file, chunkSize) {
        const chunks = [];
        const totalChunks = Math.ceil(file.size / chunkSize);
        for (let i = 0; i < totalChunks; i++) {
            const start = i * chunkSize;
            const end = Math.min(start + chunkSize, file.size);
            chunks.push(file.slice(start, end));
        }
        return {chunks, totalChunks};
    }

    /**
     * 初始化上传任务
     */
    async initTask(file, extraParams = {}) {
        const fileMd5 = await this.calculateMD5(file);
        const needChunk = file.size > this.chunkSizeThreshold;
        const totalChunks = needChunk ? Math.ceil(file.size / this.chunkSize) : undefined;
        const chunkSize = needChunk ? this.chunkSize : undefined;

        const body = {
            originalName: file.name,
            fileSize: file.size,
            fileMd5,
            mimeType: file.type || 'application/octet-stream',
            totalChunks,
            chunkSize,
            ...extraParams,
        };

        const response = await fetch(`${this.baseURL}/init`, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                ...this._getHeaders(),
            },
            body: JSON.stringify(body),
        });

        const result = await response.json();
        if (result.code !== 200) throw new Error(result.message || '初始化上传任务失败');
        return result.data;
    }

    /**
     * 秒传检测
     */
    async quickCheck(uploadId) {
        const response = await fetch(`${this.baseURL}/${uploadId}/quick-check`, {
            method: 'POST',
            headers: this._getHeaders(),
        });
        const result = await response.json();
        if (result.code !== 200) throw new Error(result.message || '秒传检测失败');
        return result.data;
    }

    /**
     * 全量上传
     */
    async uploadFull(uploadId, file) {
        const formData = new FormData();
        formData.append('file', file);

        const response = await fetch(`${this.baseURL}/${uploadId}/file`, {
            method: 'POST',
            headers: this._getHeaders(),
            body: formData,
        });

        const result = await response.json();
        if (result.code !== 200) throw new Error(result.message || '上传失败');
        return result.data;
    }

    /**
     * 上传单个分片
     */
    async uploadChunk(uploadId, chunkNumber, chunk, chunkMd5) {
        const formData = new FormData();
        formData.append('file', chunk);
        if (chunkMd5) {
            formData.append('chunkMd5', chunkMd5);
        }

        const response = await fetch(
            `${this.baseURL}/${uploadId}/chunks/${chunkNumber}`,
            {
                method: 'POST',
                headers: this._getHeaders(),
                body: formData,
            }
        );

        const result = await response.json();
        if (result.code !== 200) throw new Error(result.message || `分片 ${chunkNumber} 上传失败`);
        return result.data;
    }

    /**
     * 完成分片上传（合并分片）
     */
    async completeUpload(uploadId) {
        const response = await fetch(`${this.baseURL}/${uploadId}/complete`, {
            method: 'POST',
            headers: this._getHeaders(),
        });

        const result = await response.json();
        if (result.code !== 200) throw new Error(result.message || '完成上传失败');
        return result.data;
    }

    /**
     * 并发控制上传分片
     */
    async uploadChunksWithConcurrency(uploadId, chunks, totalChunks, startFrom = 1) {
        let uploadedCount = startFrom - 1;
        const queue = [];

        for (let i = startFrom; i <= totalChunks; i++) {
            const chunk = chunks[i - 1];
            const task = (async () => {
                // 计算分片 MD5（可选，用于校验）
                const chunkMd5 = await this.calculateMD5(chunk);
                const result = await this.uploadChunk(uploadId, i, chunk, chunkMd5);
                uploadedCount++;
                this.onProgress({
                    uploadedChunks: uploadedCount,
                    totalChunks,
                    percent: Math.round((uploadedCount / totalChunks) * 100),
                });
                return result;
            })();

            queue.push(task);

            // 控制并发数
            if (queue.length >= this.concurrency) {
                await Promise.race(queue);
                // 移除已完成的 promise
                const settled = queue.filter((p) => {
                    let done = false;
                    p.then(() => {
                        done = true;
                    }).catch(() => {
                        done = true;
                    });
                    return done;
                });
                // 简化处理：等待最快的完成，再补充
            }

            if (queue.length >= this.concurrency) {
                await Promise.allSettled(queue.splice(0, queue.length - this.concurrency + 1));
            }
        }

        // 等待剩余任务完成
        await Promise.all(queue);
    }

    /**
     * 便捷方法：一键上传文件
     */
    async upload(file, extraParams = {}) {
        try {
            // 1. 文件大小校验
            if (file.size > this.maxFileSize) {
                throw new Error(`文件大小 ${file.size} 超过最大限制 ${this.maxFileSize}`);
            }

            // 2. 初始化
            this.onProgress({stage: 'init', percent: 0});
            const initData = await this.initTask(file, extraParams);

            // 3. 根据模式处理
            switch (initData.uploadMode) {
                case 1: {
                    // 秒传
                    if (initData.completed) {
                        this.onProgress({stage: 'done', percent: 100});
                        this.onComplete(initData);
                        return initData;
                    }
                    if (initData.quickUploadAvailable) {
                        this.onProgress({stage: 'quick-check', percent: 50});
                        const result = await this.quickCheck(initData.uploadId);
                        this.onProgress({stage: 'done', percent: 100});
                        this.onComplete(result);
                        return result;
                    }
                    // 秒传不可用，不应到达这里，但做防御处理
                    throw new Error('秒传不可用');
                }

                case 3: {
                    // 全量上传
                    this.onProgress({stage: 'uploading', percent: 30});
                    const result = await this.uploadFull(initData.uploadId, file);
                    this.onProgress({stage: 'done', percent: 100});
                    this.onComplete(result);
                    return result;
                }

                case 2: {
                    // 分片上传
                    const {chunks, totalChunks} = this.sliceFile(file, initData.chunkSize || this.chunkSize);
                    const uploadedChunks = initData.taskStatus === 1
                        ? (await this._getUploadedChunksCount(initData.uploadId))
                        : 0;

                    this.onProgress({
                        stage: 'uploading',
                        percent: Math.round((uploadedChunks / totalChunks) * 100),
                    });

                    await this.uploadChunksWithConcurrency(
                        initData.uploadId,
                        chunks,
                        totalChunks,
                        uploadedChunks + 1
                    );

                    // 完成合并
                    this.onProgress({stage: 'merging', percent: 95});
                    const result = await this.completeUpload(initData.uploadId);
                    this.onProgress({stage: 'done', percent: 100});
                    this.onComplete(result);
                    return result;
                }

                default:

    }
}

    concurrency: 3,
```
