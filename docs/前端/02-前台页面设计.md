# 前台页面设计

本文档涵盖 10 个前台用户页面，基于后端 DTO 字段定义详细描述每个页面的 UI 布局、功能列表、交互流程、对应 API、建议组件拆分及状态管理。

---

## 1.1 登录页

**路由建议**：`/login`

**UI 布局建议**：
- 居中卡片式布局，背景可使用渐变或模糊图
- 卡片内顶部显示 Logo / 站点名
- 使用 Tab 切换三种模式：账号登录、邮箱验证码登录、注册
- 底部放置"忘记密码"链接（如有）及版权信息

**功能列表**：
- 账号密码登录（支持用户名/邮箱/手机号）
- 邮箱验证码登录
- 注册新账号
- 登录/注册成功后自动跳转首页或之前的页面
- 表单校验（必填、格式校验）

**交互流程**：
1. 用户打开登录页 → 默认展示"账号登录"Tab
2. **账号登录**：填写 username + password → 点击"登录" → 调用 `POST /api/auth/login` → 成功后存储 token 并跳转首页
3. **邮箱登录**：切换至"邮箱登录"Tab → 填写 email → 点击"发送验证码" → 调用 `POST /api/auth/email-code` → 输入验证码 → 点击"登录" → 调用 `POST /api/auth/email-login`
4. **注册**：切换至"注册"Tab → 填写注册表单 → 点击"注册" → 调用 `POST /api/auth/register` → 成功后自动登录或跳转登录页

**对应 API**：

| API | 方法 | 说明 | 请求参数 | 响应字段 |
|---|---|---|---|---|
| `/api/auth/login` | POST | 账号登录 | `username`(string, 必填), `password`(string, 必填) | token, refreshToken, userInfo |
| `/api/auth/register` | POST | 注册 | `username`(string, 必填), `password`(string, 必填), `nickname`(string), `email`(string), `phone`(string) | - |
| `/api/auth/email-code` | POST | 发送邮箱验证码 | `email`(string, 必填, 邮箱格式) | - |
| `/api/auth/email-login` | POST | 邮箱验证码登录 | `email`(string, 必填, 邮箱格式), `code`(string, 必填) | token, refreshToken, userInfo |

**建议组件拆分**：
- LoginPage — 页面容器
- LoginForm — 账号密码登录表单
- EmailLoginForm — 邮箱验证码登录表单
- RegisterForm — 注册表单
- AuthTabs — Tab 切换组件

**状态管理建议**：
- 全局状态：`currentUser`（含 id, username, nickname, avatar, email, phone, status, roles, permissions）、`token`、`refreshToken`
- 页面级状态：当前 Tab、表单数据、loading 状态、验证码倒计时

## 1.2 首页文章列表

**路由建议**：`/` 或 `/home`

**UI 布局建议**：
- 顶部导航栏（Logo、搜索框、用户头像/登录按钮）
- 左侧边栏：分类树（可折叠）
- 右侧主内容区：文章卡片列表（支持分页/无限滚动）
- 右侧边栏：标签云、热门文章（可选）

**功能列表**：
- 文章卡片列表展示（封面图、标题、摘要、作者、浏览/点赞/评论/收藏数、发布时间）
- 分页加载或无限滚动
- 按分类筛选（左侧分类树点击）
- 按标签筛选（标签云点击）
- 关键词搜索
- 排序切换（最新 latest、最热）
- 置顶文章标识

**交互流程**：
1. 用户打开首页 → 并行调用 `GET /api/articles`、`GET /api/categories/tree`、`GET /api/tags` 获取数据
2. 点击分类节点 → 携带 categoryId 重新调用 `GET /api/articles` 筛选
3. 点击标签 → 携带 tagId 重新调用 `GET /api/articles` 筛选
4. 输入搜索关键词 → 携带 keyword 重新调用 `GET /api/articles` 搜索
5. 切换排序 → 携带 sort 参数重新加载
6. 点击文章卡片 → 跳转文章详情页 `/articles/:id`

**对应 API**：

| API | 方法 | 说明 | 请求参数 | 响应字段 |
|---|---|---|---|---|
| `/api/articles` | GET | 分页查询已发布文章 | `current`(long, 默认1), `size`(long, 默认10), `keyword`(string), `categoryId`(long), `tagId`(long), `sort`(string, 默认latest) | 分页列表: `id`, `title`, `summary`, `coverImage`, `authorId`, `authorName`, `isTop`, `accessLevel`, `viewCount`, `likeCount`, `commentCount`, `collectCount`, `publishTime` |
| `/api/categories/tree` | GET | 分类树 | 无 | 树结构: `id`, `parentId`, `name`, `code`, `type`, `level`, `sortOrder`, `icon`, `description`, `children[]` |
| `/api/tags` | GET | 标签列表 | 无 | 列表: `id`, `name`, `color` |

**建议组件拆分**：
- HomePage — 页面容器
- ArticleCardList — 文章卡片列表（含分页）
- ArticleCard — 单个文章卡片
- CategoryTree — 左侧分类树（可折叠）
- TagCloud — 标签云
- SearchBar — 顶部搜索栏
- SortSelector — 排序选择器

**状态管理建议**：
- 页面级状态：文章分页列表、分类树数据、标签列表、当前筛选条件（keyword, categoryId, tagId, sort）、分页信息
- 全局状态：currentUser（判断登录状态，显示头像或登录按钮）

## 1.3 文章详情页

**路由建议**：`/articles/:id`

**UI 布局建议**：
- 顶部面包屑导航（首页 > 分类 > 文章标题）
- 左侧主内容区（70%宽度）：文章标题 + 元信息 + Markdown 渲染内容
- 右侧边栏（30%宽度）：目录导航（TOC）、文章标签
- 底部评论区：评论输入框 + 树形评论列表
- 浮动操作栏：点赞、收藏、返回顶部

**功能列表**：
- 文章内容 Markdown 渲染
- 文章元信息展示（作者名称/头像、发布时间、浏览/点赞/评论/收藏数）
- 分类面包屑、标签展示
- 点赞/取消点赞
- 收藏/取消收藏（选择收藏夹）
- 评论区（树形评论、发表评论、回复评论、点赞评论、删除自己的评论）
- 密码访问验证（accessLevel=2 时弹出密码输入框）
- 自动记录浏览足迹

**交互流程**：
1. 用户进入文章详情 → 调用 `GET /api/articles/{id}` 获取文章详情（含是否已点赞/已收藏）
2. 页面渲染 Markdown 内容 → 根据 accessLevel 判断是否弹出密码输入框
3. 用户点击"点赞" → 调用 `POST /api/user/articles/{id}/likes` → 更新点赞状态和计数
4. 用户点击"取消点赞" → 调用 `DELETE /api/user/articles/{id}/likes` → 更新状态
5. 用户点击"收藏" → 弹出收藏夹选择 → 调用 `POST /api/user/collections` 添加收藏
6. 用户点击"取消收藏" → 调用 `DELETE /api/user/collections/{id}`
7. 加载评论区 → 调用 `GET /api/comments`（targetType=article, targetId=文章ID）
8. 用户发表评论 → 调用 `POST /api/user/comments` → 刷新评论列表
9. 用户回复评论 → 填写回复内容 → 调用 `POST /api/user/comments`（携带 rootId/parentId）
10. 用户点赞评论 → 调用 `POST /api/user/comments/{id}/likes`
11. 用户删除自己的评论 → 调用 `DELETE /api/user/comments/{id}`

**对应 API**：

| API | 方法 | 说明 | 请求参数 | 响应字段 |
|---|---|---|---|---|
| `/api/articles/{id}` | GET | 文章详情 | path: id | `id`, `title`, `summary`, `content`, `coverImage`, `authorId`, `authorName`, `isTop`, `isOriginal`, `sourceUrl`, `accessLevel`, `viewCount`, `likeCount`, `commentCount`, `collectCount`, `shareCount`, `publishTime`, `categories[]`, `tags[]`, `liked`, `collected`, `canComment` |
| `/api/user/articles/{id}/likes` | POST | 点赞 | path: id | - |
| `/api/user/articles/{id}/likes` | DELETE | 取消点赞 | path: id | - |
| `/api/comments` | GET | 评论树 | `current`, `size`, `targetType`(string, 必填), `targetId`(long, 必填) | 树结构: `id`, `targetId`, `targetType`, `content`, `images[]`, `userId`, `userNickname`, `userAvatar`, `rootId`, `parentId`, `likeCount`, `replyCount`, `status`, `createdAt`, `liked`, `children[]` |
| `/api/user/comments` | POST | 发表评论 | `targetType`(string, 必填), `targetId`(long, 必填), `content`(string, 必填), `images[]`, `rootId`(long, 默认0), `parentId`(long, 默认0) | - |
| `/api/user/comments/{id}` | DELETE | 删除评论 | path: id | - |
| `/api/user/comments/{id}/likes` | POST | 点赞评论 | path: id | - |
| `/api/user/comments/{id}/likes` | DELETE | 取消点赞评论 | path: id | - |
| `/api/user/collections` | POST | 收藏 | `folderId`(long), `targetId`(long, 必填), `targetType`(string, 必填), `remark`(string) | - |
| `/api/user/collections/{id}` | DELETE | 取消收藏 | path: id | - |

**建议组件拆分**：
- ArticleDetailPage — 页面容器
- ArticleHeader — 文章标题、元信息、标签、面包屑
- ArticleContent — Markdown 内容渲染区
- ArticleSidebar — 右侧目录导航（TOC）
- ActionBar — 浮动操作栏（点赞、收藏、分享）
- CommentSection — 评论区容器
- CommentInput — 评论输入框（支持图片上传）
- CommentList — 树形评论列表
- CommentItem — 单条评论（含回复按钮、点赞按钮、删除按钮）
- PasswordModal — 密码访问验证弹窗
- CollectionModal — 收藏夹选择弹窗

**状态管理建议**：
- 页面级状态：文章详情数据、评论列表、是否已点赞、是否已收藏、loading 状态
- 全局状态：currentUser（判断是否登录，是否可以评论/点赞/收藏）

## 1.4 个人中心

**路由建议**：`/user/profile`

**UI 布局建议**：
- 左侧个人信息卡片（头像、昵称、关注/粉丝统计）
- 右侧 Tab 切换区域：关注列表、粉丝列表
- 顶部操作栏：编辑个人信息（如有）、关注统计

**功能列表**：
- 个人信息展示（头像、用户名、昵称、邮箱、手机、角色等）
- 关注/粉丝数统计展示
- 关注列表（分页，支持"只看特别关注"筛选）
- 粉丝列表（分页）
- 互关状态展示
- 关注/取消关注操作
- 设置/取消特别关注
- 编辑关注备注

**交互流程**：
1. 用户进入个人中心 → 并行调用 `GET /api/auth/current-user`、`GET /api/user/follows/count` 获取基本信息
2. 默认展示"关注列表"Tab → 调用 `GET /api/user/follows` 获取关注列表
3. 切换到"粉丝列表"Tab → 调用 `GET /api/user/fans` 获取粉丝列表
4. 点击"只看特别关注"开关 → 调用 `GET /api/user/follows`（specialOnly=true）
5. 点击"关注/取消关注"按钮 → 调用 `POST/DELETE /api/user/follows/{userId}` → 刷新列表和计数
6. 点击"特别关注"星标 → 调用 `PUT /api/user/follows/{userId}/special`（specialFollow=1/0）
7. 点击"编辑备注" → 弹出输入框 → 调用 `PUT /api/user/follows/{userId}/remark`

**对应 API**：

| API | 方法 | 说明 | 请求参数 | 响应字段 |
|---|---|---|---|---|
| `/api/auth/current-user` | GET | 当前用户信息 | 无 | `id`, `username`, `nickname`, `avatar`, `email`, `phone`, `status`, `roles[]`, `permissions[]` |
| `/api/user/follows` | GET | 我的关注列表 | `current`, `size`, `specialOnly`(boolean) | 分页列表: `relationId`, `userId`, `username`, `nickname`, `avatar`, `isSpecialFollow`, `remark`, `mutualFollow`, `followTime` |
| `/api/user/fans` | GET | 我的粉丝列表 | `current`, `size` | 分页列表: `relationId`, `userId`, `username`, `nickname`, `avatar`, `isSpecialFollow`, `remark`, `mutualFollow`, `followTime` |
| `/api/user/follows/count` | GET | 关注/粉丝统计 | 无 | `followingCount`, `fanCount` |
| `/api/user/follows/mutual` | GET | 互关状态 | targetUserId | `targetUserId`, `following`, `followedBy`, `mutualFollow` |
| `/api/user/follows/{userId}` | POST | 关注 | path: userId | - |
| `/api/user/follows/{userId}` | DELETE | 取消关注 | path: userId | - |
| `/api/user/follows/{userId}/special` | PUT | 特别关注 | path: userId, `specialFollow`(int, 必填, 0/1) | - |
| `/api/user/follows/{userId}/remark` | PUT | 关注备注 | path: userId, `remark`(string, 最大256字符) | - |

**建议组件拆分**：
- ProfilePage — 页面容器
- ProfileCard — 个人信息卡片（头像、昵称、统计数）
- FollowStats — 关注/粉丝数统计组件
- FollowList — 关注列表
- FanList — 粉丝列表
- FollowUserItem — 用户列表项（含关注/取消关注、特别关注、备注按钮）
- RemarkEditModal — 备注编辑弹窗

**状态管理建议**：
- 全局状态：currentUser
- 页面级状态：关注列表、粉丝列表、关注/粉丝计数、当前 Tab、分页信息

## 1.5 我的收藏

**路由建议**：`/user/collections`

**UI 布局建议**：
- 左侧收藏夹列表（可点击切换，默认收藏夹置顶）
- 右侧主内容区：当前收藏夹内的收藏记录列表
- 顶部操作栏：新建收藏夹、编辑收藏夹、删除收藏夹

**功能列表**：
- 收藏夹列表展示（名称、类型、收藏数量、是否公开标识）
- 新建收藏夹
- 编辑收藏夹（名称、描述、公开状态等）
- 删除收藏夹
- 收藏夹内收藏记录列表（分页，展示目标标题、目标地址、备注、收藏时间）
- 添加收藏（从文章详情页触发）
- 删除收藏

**交互流程**：
1. 用户进入收藏页 → 调用 `GET /api/user/collection-folders` 获取收藏夹列表
2. 默认选中第一个收藏夹 → 调用 `GET /api/user/collections`（folderId=当前收藏夹ID）获取收藏记录
3. 点击其他收藏夹 → 更新 folderId 重新加载收藏记录
4. 点击"新建收藏夹" → 弹出表单 → 调用 `POST /api/user/collection-folders` → 刷新收藏夹列表
5. 点击"编辑收藏夹" → 弹出编辑表单 → 调用 `PUT /api/user/collection-folders/{id}` → 刷新列表
6. 点击"删除收藏夹" → 确认弹窗 → 调用 `DELETE /api/user/collection-folders/{id}` → 刷新
7. 点击收藏记录的"删除" → 调用 `DELETE /api/user/collections/{id}` → 刷新收藏记录列表

**对应 API**：

| API | 方法 | 说明 | 请求参数 | 响应字段 |
|---|---|---|---|---|
| `/api/user/collection-folders` | GET | 收藏夹列表 | 无 | 列表: `id`, `userId`, `folderName`, `folderType`, `description`, `isPublic`, `isDefault`, `sortOrder`, `collectionCount`, `createdAt`, `updatedAt` |
| `/api/user/collection-folders` | POST | 新建收藏夹 | `folderName`(string, 必填), `folderType`(string), `description`(string), `isPublic`(int), `isDefault`(int), `sortOrder`(int) | - |
| `/api/user/collection-folders/{id}` | PUT | 编辑收藏夹 | path: id, 同上字段 | - |
| `/api/user/collection-folders/{id}` | DELETE | 删除收藏夹 | path: id | - |
| `/api/user/collections` | GET | 收藏列表 | `current`, `size`, `folderId`(long) | 分页列表: `id`, `folderId`, `targetId`, `targetType`, `remark`, `targetTitle`, `targetUrl`, `createdAt` |
| `/api/user/collections` | POST | 添加收藏 | `folderId`(long), `targetId`(long, 必填), `targetType`(string, 必填), `remark`(string) | - |
| `/api/user/collections/{id}` | DELETE | 删除收藏 | path: id | - |

**建议组件拆分**：
- CollectionsPage — 页面容器
- FolderList — 收藏夹列表（左侧）
- FolderItem — 收藏夹列表项
- CollectionRecordList — 收藏记录列表（右侧）
- CollectionRecordItem — 收藏记录项
- FolderFormModal — 收藏夹新增/编辑弹窗
- AddCollectionModal — 添加收藏弹窗（选择收藏夹）

**状态管理建议**：
- 页面级状态：收藏夹列表、当前选中收藏夹、收藏记录列表、分页信息、弹窗显示状态

## 1.6 我的足迹

**路由建议**：`/user/footprints`

**UI 布局建议**：
- 顶部操作栏：按目标类型筛选下拉框、清空全部按钮
- 主内容区：足迹时间线列表（分页）

**功能列表**：
- 浏览足迹列表（分页展示，按时间倒序）
- 按目标类型筛选（如 article 等）
- 点击足迹跳转到目标页面
- 删除单条足迹
- 清空全部足迹

**交互流程**：
1. 用户进入足迹页 → 调用 `GET /api/user/footprints` 获取足迹列表
2. 选择目标类型筛选 → 携带 targetType 重新加载
3. 点击足迹记录 → 跳转到 targetUrl 对应页面
4. 点击单条"删除" → 调用 `DELETE /api/user/footprints/{id}` → 刷新列表
5. 点击"清空全部" → 确认弹窗 → 调用 `DELETE /api/user/footprints` → 清空列表

**对应 API**：

| API | 方法 | 说明 | 请求参数 | 响应字段 |
|---|---|---|---|---|
| `/api/user/footprints` | GET | 足迹列表 | `current`(long, 默认1), `size`(long, 默认10), `targetType`(string) | 分页列表: `id`, `targetId`, `targetType`, `targetTitle`, `targetUrl`, `visitedAt` |
| `/api/user/footprints/{id}` | DELETE | 删除足迹 | path: id | - |
| `/api/user/footprints` | DELETE | 清空足迹 | 无 | - |

**建议组件拆分**：
- FootprintsPage — 页面容器
- FootprintFilter — 筛选工具栏（目标类型选择）
- FootprintTimeline — 足迹时间线列表
- FootprintItem — 单条足迹（标题、访问时间、删除按钮）
- ConfirmModal — 清空确认弹窗

**状态管理建议**：
- 页面级状态：足迹列表、分页信息、当前筛选条件（targetType）、loading 状态

## 1.8 聊天页

**路由建议**：`/chat`

**UI 布局建议**：
- 三栏布局（参考主流 IM 应用）
- 左侧边栏：会话列表（搜索框 + 会话列表项，含未读数、最后消息预览）

**功能列表**：

**会话列表**：
- 会话列表展示（单聊/群聊，未读数，最后消息预览）

- 打开单聊（创建/获取单聊会话）
- 创建群聊

**聊天主窗口**：
- 消息历史（分页向上加载）
- 发送文本消息（支持回复引用）
- 发送图片/文件消息
- 编辑已发送消息
- 撤回消息
- 删除消息（仅隐藏自己的视图）
- 已读回执（投递/已读状态）
- WebSocket 实时通信

**群聊管理**：
- 群详情（群名称、群头像、群公告）
- 群成员列表
- 邀请成员
- 设置/取消管理员
- 转让群主
- 禁言成员
- 更新群公告
- 移除成员
- 退出群聊
- 解散群聊（仅群主）

**交互流程**：
1. 用户进入聊天页 → 建立 WebSocket 连接 → 发送 ready 信令 → 调用 `GET /api/user/chat/conversations` 获取会话列表
2. 点击会话 → 调用 `GET /api/user/chat/conversations/{id}` 获取会话详情 → 调用 `GET /api/user/chat/conversations/{id}/messages` 加载消息历史
3. 输入文本并发送 → 调用 `POST /api/user/chat/messages/text` → WebSocket 推送新消息
4. 上传文件并发送 → 先上传文件获取 businessId → 调用 `POST /api/user/chat/messages/file`
5. 回复消息 → 点击回复按钮 → 输入框携带 replyMessageId → 发送
6. 编辑消息 → 右键/长按 → 编辑模式 → 调用 `PUT /api/user/chat/messages/{id}`
7. 撤回消息 → 调用 `POST /api/user/chat/messages/{id}/revoke`
8. 删除消息 → 调用 `DELETE /api/user/chat/messages/{id}`
9. 滚动加载历史 → 携带 beforeMessageId 调用 `GET .../messages` 向上加载
10. 已读推进 → 调用 `POST /api/user/chat/conversations/{id}/read`（readMessageId）
11. 点击"创建群聊" → 弹窗填写群名称、选择成员 → 调用 `POST /api/user/chat/groups`
12. 打开单聊 → 调用 `POST /api/user/chat/single-conversations`（targetUserId）
13. 群管理操作 → 各自调用对应 API

## 1.10 我的文件

**路由建议**：`/user/files`

**UI 布局建议**：
- 顶部操作栏：上传按钮、筛选条件（文件名关键字、文件状态、业务分类、引用类型）
- 主内容区上方：文件卡片/列表视图切换
- 主内容区：文件列表（分页）
- 右侧或底部抽屉：上传任务列表（查看上传进度）

**功能列表**：
- 文件列表展示（分页，展示文件名、文件地址、大小、类型、状态、创建时间等）
- 按文件名搜索
- 按状态/业务分类/引用类型筛选
- 上传文件（支持三种模式）：
  - 普通上传：小文件直接上传
  - 分片上传：大文件分片上传，支持进度显示
  - 秒传：基于 MD5 检测已有文件快速引用
- 查看上传任务列表及进度
- 删除文件引用

**交互流程**：
1. 用户进入文件页 → 调用 `GET /api/user/files` 获取文件列表
2. 筛选条件变更 → 携带筛选参数重新加载
3. 点击"上传文件" → 弹出上传弹窗 → 填写文件信息 → 调用 `POST /api/user/files/upload-tasks/init` 初始化上传任务
4. 若 quickUploadAvailable=true 且 completed=true → 秒传完成，直接刷新文件列表
5. 若为普通上传模式 → 调用 `POST /api/user/files/upload-tasks/{uploadId}/file` 上传文件
6. 若为分片上传模式 → 逐片调用 `POST /api/user/files/upload-tasks/{uploadId}/chunks/{chunkNumber}` → 完成后调用 `POST /api/user/files/upload-tasks/{uploadId}/complete`
7. 查看上传任务 → 调用 `GET /api/user/files/upload-tasks` 获取任务列表
8. 删除文件 → 调用 `DELETE /api/user/files/{businessId}` → 刷新文件列表

**对应 API**：

| API | 方法 | 说明 | 请求参数 | 响应字段 |
|---|---|---|---|---|
| `/api/user/files/upload-tasks/init` | POST | 初始化上传 | `originalName`(string, 必填), `fileSize`(long, 必填), `fileMd5`(string), `mimeType`(string), `referenceType`(string), `referenceId`(long), `category`(string), `isPublic`(int), `totalChunks`(int), `chunkSize`(long), `remark`(string) | `taskId`, `uploadId`, `uploadMode`, `quickUploadAvailable`, `completed`, `totalChunks`, `chunkSize`, `taskStatus`, `fileId`, `fileUrl`, `businessId` |
| `/api/user/files/upload-tasks/{uploadId}/quick-check` | POST | 秒传检测 | path: uploadId | 同 init 响应 |
| `/api/user/files/upload-tasks/{uploadId}/file` | POST | 普通上传 | path: uploadId, file(multipart) | `uploadId`, `taskId`, `fileId`, `businessId`, `quickUpload`, `taskStatus`, `fileUrl`, `referenceCount` |
| `/api/user/files/upload-tasks/{uploadId}/chunks/{chunkNumber}` | POST | 分片上传 | path: uploadId, path: chunkNumber, chunk(multipart) | `uploadId`, `chunkNumber`, `uploadedChunks`, `totalChunks`, `taskStatus` |
| `/api/user/files/upload-tasks/{uploadId}/complete` | POST | 完成上传 | path: uploadId | 上传结果 |
| `/api/user/files` | GET | 文件列表 | `current`, `size`, `keyword`(string), `status`(int), `category`(string), `referenceType`(string) | 分页列表: `businessId`, `fileId`, `fileName`, `originalName`, `fileUrl`, `fileSize`, `fileType`, `mimeType`, `category`, `referenceType`, `referenceId`, `isPublic`, `status`, `createdAt` |
| `/api/user/files/upload-tasks` | GET | 上传任务列表 | `current`, `size`, `taskStatus`(int), `isQuickUpload`(int), `isChunked`(int) | 分页列表: `id`, `uploadId`, `fileId`, `originalName`, `fileSize`, `isQuickUpload`, `isChunked`, `chunkSize`, `totalChunks`, `uploadedChunks`, `taskStatus`, `errorCode`, `errorMessage`, `startTime`, `completeTime`, `createdAt` |
