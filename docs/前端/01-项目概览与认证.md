### 1.3 统一响应结构

所有接口返回 `Result<T>` 统一响应体，包含 4 个字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | Integer | 业务状态码，`200` 表示成功 |
| `message` | String | 状态描述信息 |
| `timestamp` | Long | 服务器时间戳（毫秒） |
| `data` | T | 业务数据，失败时通常为 `null` |

成功响应示例：

```json
{
  "code": 200,
  "message": "成功",
  "timestamp": 1774310400000,
  "data": {
    "id": 1,
    "username": "admin"
  }
}
```

无数据返回的成功响应（如删除操作）：

```json
{
  "code": 200,
  "message": "成功",
  "timestamp": 1774310400000,
  "data": null
}
```

失败响应示例：

```json
{
  "code": 40104,
  "message": "账号已锁定",
  "timestamp": 1774310400000,
  "data": null
}
```

此外，每个响应头包含 `X-Trace-Id`，用于链路追踪，排查问题时可将此值提供给后端。

## 2. 认证机制

### 2.1 JWT Token 流程

系统采用双 Token（AccessToken + RefreshToken）机制，完整流程如下：

```
1. 用户在登录页输入账号密码
   |
   v
2. 前端调用 POST /api/auth/login
   |
   v
3. 后端验证成功，返回 AuthenticationToken：
   { tokenType: "Bearer", accessToken: "xxx", refreshToken: "yyy", expiresIn: 3600 }
   |
   v
4. 前端将 accessToken 和 refreshToken 存储到 localStorage（或 sessionStorage）
   |
   v
5. 后续每次 API 请求在 Header 中携带：
   Authorization: Bearer <accessToken>
   |
   v
6. 后端验证 accessToken：
   - 有效 → 正常处理请求，返回数据
   - 已过期 → 返回 { code: 40102, message: "未登录或登录已过期" }
   |
   v
7. 前端收到 40102：
   - 取出 refreshToken，调用 POST /api/auth/refresh
   - 成功 → 用新的 accessToken 重试原请求
   - 失败（refreshToken 也过期）→ 清除本地 Token，跳转到登录页
```

### 2.2 Token 参数说明

登录/注册/邮箱登录成功后返回 `AuthenticationToken`：

| 参数 | 类型 | 说明 |
|------|------|------|
| `tokenType` | String | 令牌类型，固定值 `"Bearer"` |
| `accessToken` | String | 访问令牌，默认有效期 **3600 秒**（1 小时） |
| `refreshToken` | String | 刷新令牌，默认有效期 **604800 秒**（7 天） |
| `expiresIn` | Integer | accessToken 剩余有效时间（秒），通常为 3600 |

Token 有效期可通过后端配置调整（`security.session.accessTokenTimeToLive` 和 `security.session.refreshTokenTimeToLive`），设置为 `-1` 表示永不过期。

## 3. Token 自动刷新（前端实现指南）

### 3.1 流程描述

```
1. 前端发起业务请求
   |
   v
2. 后端返回 { code: 40102, message: "未登录或登录已过期" }
   |
   v
3. 前端检测到 Token 过期，检查本地是否有 refreshToken
   |
   +-- 没有 refreshToken → 清除登录状态，跳转登录页
   |
   +-- 有 refreshToken →
       |
       v
4. 调用 POST /api/auth/refresh（使用 refreshToken）
   |
   +-- 刷新成功 →
   |     |
   |     v
   |   更新本地 accessToken 和 refreshToken
   |     |
   |     v
   |   使用新 Token 重试原请求（以及所有排队的请求）
   |
   +-- 刷新失败 →
         |
         v
       清除本地所有 Token
         |
         v
       跳转登录页
```

### 3.2 并发刷新处理

当多个请求同时因 Token 过期返回 40102 时，必须防止并发刷新（多次调用 refresh 接口会导致第一次之后的 refreshToken 失效）。推荐使用 Promise 队列机制：第一个 40102 触发刷新，后续 40102 请求排队等待同一个刷新 Promise 完成。

### 3.3 伪代码示例（Axios 拦截器）

```javascript
import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:8000',
});

// 是否正在刷新 Token
let isRefreshing = false;
// 等待 Token 刷新的请求队列
let refreshSubscribers = [];

// 将排队请求添加到订阅列表
function subscribeTokenRefresh(resolve) {
  refreshSubscribers.push(resolve);
}

// Token 刷新成功后，重试所有排队请求
function onTokenRefreshed(newAccessToken) {
  refreshSubscribers.forEach(resolve => resolve(newAccessToken));
  refreshSubscribers = [];
}

// Token 刷新失败，拒绝所有排队请求
function onRefreshFailed(error) {
  refreshSubscribers.forEach(resolve => resolve(null));
  refreshSubscribers = [];
}

// 请求拦截器：自动添加 Authorization 头
api.interceptors.request.use(config => {
  const accessToken = localStorage.getItem('accessToken');
  if (accessToken) {
    config.headers.Authorization = `Bearer ${accessToken}`;
  }
  return config;
});

// 响应拦截器：处理 Token 过期
api.interceptors.response.use(
  response => {
    // 正常响应直接返回
    return response;
  },
  async error => {
    const originalRequest = error.config;

    // 非 40102 错误直接抛出
    if (!error.response || error.response.data.code !== 40102) {
      return Promise.reject(error);
    }

    // 防止重复刷新（标记原请求已重试）
    if (originalRequest._retry) {
      return Promise.reject(error);
    }
    originalRequest._retry = true;

    const refreshToken = localStorage.getItem('refreshToken');
    if (!refreshToken) {
      // 没有 refreshToken，跳转登录
      clearAuthAndRedirect();
      return Promise.reject(error);
    }

    if (isRefreshing) {
      // 已有刷新请求进行中，排队等待
      return new Promise(resolve => {
        subscribeTokenRefreshed(token => {
          if (token) {
            originalRequest.headers.Authorization = `Bearer ${token}`;
            resolve(api(originalRequest));
          } else {
            clearAuthAndRedirect();
            resolve(Promise.reject(error));
          }
        });
      });
    }

    // 开始刷新
    isRefreshing = true;

    try {
      const response = await axios.post('http://localhost:8000/api/auth/refresh', {
        refreshToken: refreshToken,
      });

      if (response.data.code === 200 && response.data.data) {
        const tokenData = response.data.data;

        // 更新本地存储
        localStorage.setItem('accessToken', tokenData.accessToken);
        localStorage.setItem('refreshToken', tokenData.refreshToken);

        // 通知所有排队请求使用新 Token 重试
        onTokenRefreshed(tokenData.accessToken);

        // 重试原请求
        originalRequest.headers.Authorization = `Bearer ${tokenData.accessToken}`;
        return api(originalRequest);
      } else {
        // 刷新失败
        onRefreshFailed(null);
        clearAuthAndRedirect();
        return Promise.reject(error);
      }
    } catch (refreshError) {
      // refresh 接口本身异常（网络错误等）
      onRefreshFailed(null);
      clearAuthAndRedirect();
      return Promise.reject(refreshError);
    } finally {
      isRefreshing = false;
    }
  }
);

function clearAuthAndRedirect() {
  localStorage.removeItem('accessToken');
  localStorage.removeItem('refreshToken');
  // 跳转登录页（根据实际路由方案调整）
  window.location.href = '/login';
}

export default api;
```

### 3.4 前端存储建议

| 方案 | 适用场景 | 说明 |
|------|----------|------|
| localStorage | 长期登录 | 关闭浏览器后 Token 仍然保留 |
| sessionStorage | 临时会话 | 关闭标签页后 Token 清除 |
| Cookie（httpOnly） | 高安全要求 | 需后端配合设置，前端无法直接读取 |

建议将 `accessToken` 和 `refreshToken` 同时存储，并在应用启动时（如路由守卫中）检查 Token 是否存在以判断登录状态。

## 4. 登录失败锁定机制

### 4.1 后端规则

后端对账号密码登录（`POST /api/auth/login`）实现了失败次数限制和临时锁定：

| 配置项 | 配置 Key | 默认值 | 说明 |
|--------|----------|--------|------|
| 最大失败次数 | `auth.login-fail.max-attempts` | **5 次** | 连续登录失败达到此数值后锁定 |
| 锁定时长 | `auth.login-fail.lock-minutes` | **15 分钟** | 锁定持续时间 |

锁定规则：

- 同一账号连续登录失败达到最大次数（默认 5 次）后，账号被临时锁定。
- 锁定期间，即使密码正确也无法登录，返回 `40104`（账号已锁定）。
- 锁定过期后自动解除，可重新尝试登录。
- 登录成功后自动清除失败计数和锁定状态。
- 上述参数可通过后台系统配置（`sys_config` 表）动态调整。

### 4.2 前端处理建议

**密码错误时（code=40101）：**

- 展示错误提示："用户名或密码错误"。
- 建议在提示中附带剩余尝试次数信息（如："用户名或密码错误，还剩 X 次尝试机会"）。注意：后端当前未在错误响应中直接返回剩余次数，前端可根据本地记录或通过提示文案告知用户存在次数限制。

**账号锁定时（code=40104）：**

- 展示锁定提示："账号已锁定，请 15 分钟后重试"。
- 建议显示倒计时，增强用户体验。
- 锁定期间禁用登录按钮，防止用户频繁尝试。

**前端示例逻辑：**

```javascript
async function handleLogin(username, password) {
  try {
    const response = await api.post('/api/auth/login', { username, password });

    if (response.data.code === 200) {
      // 登录成功
      const token = response.data.data;
      localStorage.setItem('accessToken', token.accessToken);
      localStorage.setItem('refreshToken', token.refreshToken);
      // 跳转首页
      router.push('/');
    }
  } catch (error) {
    const { code, message } = error.response?.data || {};

    switch (code) {
      case 40101:
        // 密码错误
        showErrorMessage(message || '用户名或密码错误');
        // 可选：在 UI 上提示失败次数限制
        showWarning('连续失败 5 次将锁定账号 15 分钟');
        break;

      case 40104:
        // 账号锁定
        showErrorMessage('账号已锁定，请稍后重试');
        disableLoginButton();
        startLockCountdown(15 * 60); // 15 分钟倒计时
        break;

      case 40105:
        // 账号禁用
        showErrorMessage('账号已被禁用，请联系管理员');
        break;

      default:
        showErrorMessage(message || '登录失败，请重试');
    }
  }
}
```

## 6. 动态菜单与路由概述

### 6.1 菜单树结构

登录成功后，前端应调用 `GET /api/auth/current-user-menus` 获取当前用户的菜单树。返回的数据是 `AuthMenuInfo` 的树形列表，各字段含义如下：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Long | 菜单 ID |
| `parentId` | Long | 父菜单 ID（0 表示顶级） |
| `name` | String | 菜单名称（显示文本） |
| `type` | String | 菜单类型：C=目录，M=菜单，B=按钮 |
| `routeName` | String | 路由名称（对应 Vue Router 的 name） |
| `routePath` | String | 路由路径 |
| `component` | String | 组件路径（如 `system/user/index`） |
| `perm` | String | 权限标识（按钮级别，如 `sys:user:create`） |
| `visible` | Integer | 是否显示（1=显示，0=隐藏） |
| `sort` | Integer | 排序序号（升序） |
| `icon` | String | 菜单图标标识 |
| `redirect` | String | 重定向地址（如 `noRedirect`） |
| `alwaysShow` | Integer | 是否始终显示（1=始终显示，0=只有一个子菜单时合并显示） |
| `keepAlive` | Integer | 是否缓存（1=启用 keep-alive 缓存，0=不缓存） |
| `params` | Object | 路由参数（JSON 对象） |
| `children` | List\<AuthMenuInfo\> | 子菜单列表 |

### 6.2 菜单类型的前端处理

| type | 含义 | 前端处理方式 |
|------|------|-------------|
| **C**（目录） | 菜单容器 | 作为路由容器，渲染为侧边栏的一级分组。对应组件通常是 `<router-view />` 或 Layout 组件。自身不渲染具体页面内容，仅包含子路由。 |
| **M**（菜单） | 具体页面 | 对应一个实际的页面组件。前端需要根据 `component` 字段动态加载组件并注册路由。 |
| **B**（按钮） | 按钮权限 | 不参与路由注册。前端根据 `perm` 字段控制页面内按钮/操作的显示与隐藏。**注意：API 返回的菜单树已过滤掉 B 类型，按钮权限需通过 `permissions` 列表判断。** |

### 6.3 布尔字段说明

| 字段 | 值 | 说明 |
|------|----|------|
| `visible` | `1` | 菜单在侧边栏可见 |
| `visible` | `0` | 菜单隐藏（路由仍然注册，但不在侧边栏显示，适用于详情页等） |
| `alwaysShow` | `1` | 即使只有一个子菜单，也始终显示父级目录 |
| `alwaysShow` | `0` | 只有一个子菜单时，直接显示子菜单，省略父级 |
| `keepAlive` | `1` | 页面组件启用 keep-alive 缓存，切换页面时保留状态 |
| `keepAlive` | `0` | 不缓存，每次进入页面重新渲染 |

### 6.4 前端动态路由注册流程

```
1. 用户登录成功
   |
   v
2. 调用 GET /api/auth/current-user-menus
   |
   v
3. 遍历菜单树，过滤 type=C 和 type=M 的节点
   |
   v
4. 对每个 M 类型节点，根据 component 字段动态 import 组件
   例：component = "system/user/index" → import("@/views/system/user/index.vue")
   |
   v
5. 使用 router.addRoute() 注册路由
   - C 类型：注册为父路由，path = routePath，component = Layout
   - M 类型：注册为子路由，path = routePath，component = 动态加载的组件
   |
   v
6. 渲染侧边栏时，根据菜单树和 visible 字段生成菜单项
   - 根据 sort 字段排序
   - 根据 icon 字段渲染图标
   - 根据 alwaysShow 决定是否始终显示父级
   |
   v
7. 按钮权限控制
   - 调用 GET /api/auth/current-user 获取 permissions 列表
   - 页面中使用 v-if="hasPermission('sys:user:create')" 控制按钮显示
```

### 6.5 动态路由注册示例（伪代码）

```javascript
// 将后端菜单树转为 Vue Router 路由配置
function buildRoutes(menuTree) {
  const routes = [];

  for (const menu of menuTree) {
    if (menu.type === 'C') {
      // 目录 → Layout 路由
      const route = {
        path: menu.routePath,
        name: menu.routeName,
        component: () => import('@/layouts/DefaultLayout.vue'),
        redirect: menu.redirect === 'noRedirect' ? undefined : menu.redirect,
        meta: {
          title: menu.name,
          icon: menu.icon,
          alwaysShow: menu.alwaysShow === 1,
          hidden: menu.visible !== 1,
        },
        children: menu.children ? buildRoutes(menu.children) : [],
      };
      routes.push(route);
    } else if (menu.type === 'M') {
      // 菜单 → 具体页面
      const route = {
        path: menu.routePath,
        name: menu.routeName,
        component: () => import(`@/views/${menu.component}.vue`),
        meta: {
          title: menu.name,
          icon: menu.icon,
          keepAlive: menu.keepAlive === 1,
          hidden: menu.visible !== 1,
          params: menu.params,
        },
      };
      routes.push(route);
    }
    // B（按钮）类型不注册路由
  }

  return routes;
}

// 注册路由
const menuRoutes = buildRoutes(menuData);
menuRoutes.forEach(route => {
  router.addRoute(route);
});
```
