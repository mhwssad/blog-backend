# 08 - 动态菜单与权限路由

## 1. 流程概述

后台管理系统的菜单和路由由服务端动态下发，前端根据返回的菜单树动态生成路由和侧边栏。

整体流程：

```
用户登录
  │
  ├──→ 保存 accessToken
  │
  ├──→ GET /api/user/auth/menus
  │       ↓
  │    获取菜单树 (List<AuthMenuInfo>)
  │       ↓
  │    递归遍历菜单树
  │       ├── type=C (目录) → 生成 Layout 父路由
  │       ├── type=M (菜单) → 动态 import 组件，注册子路由
  │       └── type=B (按钮) → 提取 perm 权限标识，注册到权限集合
  │       ↓
  │    router.addRoute() 动态注册
  │       ↓
  │    渲染侧边栏菜单
  │
  └──→ 进入首页
```

## 2. AuthMenuInfo 字段说明

### 完整字段表（16 个字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Long | 菜单 ID，唯一标识 |
| `parentId` | Long | 父菜单 ID，顶级菜单为 0 或 null |
| `name` | String | 菜单名称，用于侧边栏显示和路由 meta |
| `type` | String | 菜单类型：`C`=目录，`M`=菜单，`B`=按钮 |
| `routeName` | String | 路由名称，对应 Vue Router 的 name |
| `routePath` | String | 路由路径，如 `/system/user` |
| `component` | String | 组件路径，如 `system/user/index`，对应 `@views/` 下的文件 |
| `perm` | String | 权限标识，如 `system:user:list`，用于按钮级权限控制 |
| `visible` | Integer | 是否显示：`0`=隐藏，`1`=显示 |
| `sort` | Integer | 排序号，数值越小越靠前 |
| `icon` | String | 菜单图标名称，如 `user`、`setting` |
| `redirect` | String | 跳转地址，目录类型重定向到第一个子菜单 |
| `alwaysShow` | Integer | 始终显示：`0`=否，`1`=是（只有一个子菜单时也显示为嵌套目录） |
| `keepAlive` | Integer | 缓存：`0`=不缓存，`1`=缓存（使用 KeepAlive） |
| `params` | Object | 路由参数，传递给组件的额外参数 |
| `children` | List\<AuthMenuInfo\> | 子菜单列表，递归结构 |

### type 值域与前端处理方式

| type | 值 | 含义 | 前端处理 |
|------|-----|------|----------|
| `C` | 目录 (Catalog) | 一级或二级目录容器 | 生成 Layout 包裹的父路由，渲染为侧边栏分组 |
| `M` | 菜单 (Menu) | 具体页面菜单 | 动态 `import()` 加载组件文件，注册为子路由 |
| `B` | 按钮 (Button) | 页面内操作按钮 | 不生成路由，提取 `perm` 字段注册到权限集合，用于 `v-perm` 指令 |

### 布尔字段含义

| 字段 | 值 | 含义 |
|------|-----|------|
| `visible` | `0` | 在侧边栏中隐藏（路由仍可访问，用于详情页等） |
| `visible` | `1` | 在侧边栏中正常显示 |
| `alwaysShow` | `0` | 只有一个子菜单时直接显示子菜单（不嵌套目录） |
| `alwaysShow` | `1` | 即使只有一个子菜单也显示为嵌套目录 |
| `keepAlive` | `0` | 切换路由时销毁组件，不缓存状态 |
| `keepAlive` | `1` | 使用 `<KeepAlive>` 缓存组件状态 |

## 3. 路由生成逻辑

```javascript
import Layout from '@/layout/index.vue';

/**
 * 根据菜单树动态生成 Vue Router 路由
 *
 * @param {Array} menuTree - 服务端返回的 AuthMenuInfo 菜单树
 * @returns {Array} Vue Router 路由配置数组
 */
function generateRoutes(menuTree) {
  const routes = [];
  const permissions = new Set();

  // 组件缓存映射，避免重复 import
  const componentModules = import.meta.glob('@/views/**/*.vue');

  /**
   * 递归处理菜单节点
   *
   * @param {Object} menu - 单个菜单节点
   * @param {Object|null} parentRoute - 父级路由对象
   */
  function processMenu(menu, parentRoute = null) {
    switch (menu.type) {
      case 'C': {
        // 目录类型：创建 Layout 父路由
        const route = {
          path: menu.routePath || `/${menu.id}`,
          name: menu.routeName || `Catalog_${menu.id}`,
          component: Layout,
          redirect: menu.redirect || findFirstChildPath(menu),
          meta: {
            title: menu.name,
            icon: menu.icon,
            hidden: menu.visible === 0,
            alwaysShow: menu.alwaysShow === 1,
            sort: menu.sort,
            params: menu.params,
          },
          children: [],
        };

        // 递归处理子菜单
        if (menu.children && menu.children.length > 0) {
          const sortedChildren = [...menu.children].sort(
            (a, b) => (a.sort || 0) - (b.sort || 0)
          );
          sortedChildren.forEach((child) => {
            processMenu(child, route);
          });
        }

        // 如果目录下没有可见子菜单且 alwaysShow=0，则不添加
        if (route.children.length > 0 || menu.alwaysShow === 1) {
          routes.push(route);
        }
        break;
      }

      case 'M': {
        // 菜单类型：动态加载组件
        const componentPath = menu.component
          ? `/src/views/${menu.component}.vue`
          : null;

        const childRoute = {
          path: parentRoute
            ? menu.routePath
            : menu.routePath || `/${menu.id}`,
          name: menu.routeName || `Menu_${menu.id}`,
          component: componentPath
            ? componentModules[componentPath] ||
              (() => import(`@/views/${menu.component}.vue`))
            : () => import('@/views/error/404.vue'),
          meta: {
            title: menu.name,
            icon: menu.icon,
            hidden: menu.visible === 0,
            keepAlive: menu.keepAlive === 1,
            sort: menu.sort,
            params: menu.params,
            activeMenu: parentRoute ? parentRoute.path : undefined,
          },
        };

        if (parentRoute) {
          // 作为子路由添加到父级
          parentRoute.children.push(childRoute);
        } else {
          // 没有父级，直接作为顶级路由（用 Layout 包裹）
          routes.push({
            path: '/',
            component: Layout,
            children: [childRoute],
          });
        }
        break;
      }

      case 'B': {
        // 按钮类型：提取权限标识，不生成路由
        if (menu.perm) {
          permissions.add(menu.perm);
        }
        break;
      }

      default:
        console.warn(`未知的菜单类型: ${menu.type}`, menu);
    }
  }

  // 按排序处理顶级菜单
  const sortedTree = [...menuTree].sort(
    (a, b) => (a.sort || 0) - (b.sort || 0)
  );
  sortedTree.forEach((menu) => processMenu(menu));

  return { routes, permissions };
}

/**
 * 查找目录下第一个子菜单的路径
 */
function findFirstChildPath(catalogMenu) {
  if (!catalogMenu.children || catalogMenu.children.length === 0) {
    return null;
  }
  const firstChild = catalogMenu.children
    .filter((m) => m.visible === 1)
    .sort((a, b) => (a.sort || 0) - (b.sort || 0))[0];
  return firstChild ? firstChild.routePath : null;
}

// ========== 注册路由到 Vue Router ==========

import { createRouter, createWebHistory } from 'vue-router';

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/login',
      name: 'Login',
      component: () => import('@/views/login/index.vue'),
      meta: { title: '登录', hidden: true },
    },
    {
      path: '/404',
      name: 'NotFound',
      component: () => import('@/views/error/404.vue'),
      meta: { title: '404', hidden: true },
    },
  ],
});

/**
 * 动态加载菜单并注册路由
 */
async function setupDynamicRoutes() {
  const token = localStorage.getItem('accessToken');
  if (!token) return;

  try {
    const response = await fetch('/api/user/auth/menus', {
      headers: { Authorization: `Bearer ${token}` },
    });
    const result = await response.json();

    if (result.code !== 200) {
      console.error('获取菜单失败:', result.message);
      return;
    }

    const menuTree = result.data;
    const { routes, permissions } = generateRoutes(menuTree);

    // 注册动态路由
    routes.forEach((route) => {
      router.addRoute(route);
    });

    // 添加兜底 404 路由（必须在动态路由之后注册）
    router.addRoute({
      path: '/:pathMatch(.*)*',
      redirect: '/404',
    });

    // 存储权限集合到全局状态
    window.__permissions__ = permissions;

    console.log(`已注册 ${routes.length} 个动态路由`);
    console.log(`已收集 ${permissions.size} 个权限标识`, [...permissions]);
  } catch (error) {
    console.error('加载动态菜单失败:', error);
  }
}
```

## 4. 权限指令实现

```vue
<!-- v-perm 自定义指令：按钮级权限控制 -->

<script>
/**
 * Vue 3 自定义指令 v-perm
 *
 * 用法：
 *   <button v-perm="'system:user:add'">新增用户</button>
 *   <button v-perm="['system:user:edit', 'system:user:delete']">操作</button>
 *
 * 权限判断逻辑：
 *   - 字符串：检查当前用户是否拥有该权限
 *   - 数组：检查是否拥有数组中任意一个权限（OR 关系）
 */
export const permDirective = {
  mounted(el, binding) {
    const { value } = binding;

    if (!value) return;

    const permissions = window.__permissions__ || new Set();

    let hasPermission = false;

    if (typeof value === 'string') {
      hasPermission = permissions.has(value);
    } else if (Array.isArray(value)) {
      hasPermission = value.some((perm) => permissions.has(perm));
    } else {
      console.warn('v-perm 的值应为字符串或字符串数组');
      return;
    }

    if (!hasPermission) {
      // 无权限：移除 DOM 元素
      el.parentNode && el.parentNode.removeChild(el);
    }
  },
};

/**
 * 权限检查函数（用于 JS 逻辑判断）
 *
 * @param {string|string[]} permission - 权限标识
 * @returns {boolean}
 */
export function hasPermission(permission) {
  const permissions = window.__permissions__ || new Set();

  if (typeof permission === 'string') {
    return permissions.has(permission);
  }

  if (Array.isArray(permission)) {
    return permission.some((p) => permissions.has(p));
  }

  return false;
}

// 在 main.js 中注册
// app.directive('perm', permDirective);
</script>

<!-- 使用示例 -->
<template>
  <div>
    <!-- 单个权限控制 -->
    <button v-perm="'system:user:add'" @click="handleAdd">
      新增用户
    </button>

    <!-- 多个权限（满足任一即可） -->
    <button v-perm="['system:user:edit', 'system:user:delete']">
      操作
    </button>

    <!-- 使用函数在模板中判断 -->
    <template v-if="hasPermission('system:user:export')">
      <button @click="handleExport">导出</button>
    </template>
  </div>
</template>

  // ...
}

  // ...
}
</script>
```

## 5. 路由守卫设计

```javascript
import { setupDynamicRoutes } from '@/router/dynamicRoutes';

// 标记是否已加载动态路由
let dynamicRoutesLoaded = false;

/**
 * 全局前置守卫
 */
router.beforeEach(async (to, from, next) => {
  const token = localStorage.getItem('accessToken');

  // 1. 未登录处理
  if (!token) {
    // 白名单路由，无需登录即可访问
    const whiteList = ['/login', '/404'];
    if (whiteList.includes(to.path)) {
      return next();
    }
    // 重定向到登录页，记录原始目标路径
    return next({
      path: '/login',
      query: { redirect: to.fullPath },
    });
  }

  // 2. 已登录，访问登录页，重定向到首页
  if (to.path === '/login') {
    return next({ path: '/' });
  }

  // 3. 动态路由尚未加载
  if (!dynamicRoutesLoaded) {
    try {
      await setupDynamicRoutes();
      dynamicRoutesLoaded = true;

      // 重新导航到目标路由（因为首次 addRoute 后当前导航可能还没匹配到）
      return next({ ...to, replace: true });
    } catch (error) {
      // 加载失败（Token 可能已过期）
      console.error('动态路由加载失败:', error);
      localStorage.removeItem('accessToken');
      dynamicRoutesLoaded = false;
      return next({
        path: '/login',
        query: { redirect: to.fullPath },
      });
    }
  }

  // 4. 检查目标路由权限
  if (to.matched.length === 0) {
    // 未匹配到任何路由 → 404
    return next('/404');
  }

  // 5. 检查路由 meta 中的权限要求
  const requiredPerm = to.meta?.perm;
  if (requiredPerm) {
    const permissions = window.__permissions__ || new Set();
    if (!permissions.has(requiredPerm)) {
      // 无权限访问该路由
      console.warn(`无权限访问: ${to.path}，需要权限: ${requiredPerm}`);
      return next('/404');
    }
  }

  // 6. 放行
  next();
});

/**
 * 全局后置守卫
 */
router.afterEach((to) => {
  // 更新页面标题
  const title = to.meta?.title;
  document.title = title ? `${title} - 博客后台管理` : '博客后台管理';
});

/**
 * 重置路由（退出登录时调用）
 */
export function resetRouter() {
  const currentRoutes = router.getRoutes();
  currentRoutes.forEach((route) => {
    if (route.name && !['Login', 'NotFound'].includes(route.name)) {
      router.removeRoute(route.name);
    }
  });
  dynamicRoutesLoaded = false;
  window.__permissions__ = new Set();
}

/**
 * 退出登录
 */
async function handleLogout() {
  localStorage.removeItem('accessToken');
  resetRouter();
  router.push('/login');
}
```

```vue
<template>
  <div class="sidebar">
    <el-menu
      :default-active="activeMenu"

      text-color="#bfcbd9"
      active-text-color="#409eff"

        :menu="menu"
        :base-path="menu.routePath"

<script setup>
import { computed } from 'vue';
import { useRoute } from 'vue-router';
import SidebarItem from './SidebarItem.vue';

const props = defineProps({
  menuTree: { type: Array, default: () => [] },
  isCollapsed: { type: Boolean, default: false },
});

const route = useRoute();

// 当前激活的菜单
const activeMenu = computed(() => route.path);

// 过滤隐藏菜单并排序
const visibleMenuTree = computed(() => {
  return filterVisibleMenus(props.menuTree);
});

/**
 * 递归过滤：visible=0 的菜单不显示
 */
function filterVisibleMenus(menus) {
  return menus
    .filter((m) => m.visible !== 0)
    .sort((a, b) => (a.sort || 0) - (b.sort || 0))
    .map((m) => {
      if (m.children && m.children.length > 0) {
        return {
          ...m,
          children: filterVisibleMenus(m.children),
        };
      }
      return m;
    })
    .filter((m) => {
      // 目录类型：有可见子菜单时才显示（除非 alwaysShow=1）
      if (m.type === 'C') {
        const hasVisibleChildren =
          m.children && m.children.length > 0;
        return hasVisibleChildren || m.alwaysShow === 1;
      }
      return true;
    });
}

/**
 * 图标映射：将菜单 icon 字符串映射到组件
 */
const iconMap = {
  user: 'UserFilled',
  setting: 'Setting',
  dashboard: 'Odometer',
  article: 'Document',
  comment: 'ChatDotRound',
  file: 'FolderOpened',
  log: 'Tickets',
  monitor: 'Monitor',
};

function getIconName(icon) {
  return iconMap[icon] || icon || 'Menu';
}
</script>
```

```vue
<!-- SidebarItem.vue - 递归菜单项 -->
<template>
  <!-- 只有一个子菜单且 alwaysShow=0：直接显示子菜单 -->
  <template v-if="isSingleChild && !menu.alwaysShow">
    <el-menu-item :index="singleChildFullPath">
      <el-icon><component :is="getIconName(singleChild.icon || menu.icon)" /></el-icon>

    </el-menu-item>
  </template>

  <!-- 多个子菜单或 alwaysShow=1：显示为子菜单组 -->
  <el-sub-menu v-else :index="menu.routePath">
    <template #title>
      <el-icon><component :is="getIconName(menu.icon)" /></el-icon>
      <span>{{ menu.name }}</span>
    </template>

    <!-- 递归渲染子菜单 -->
    <template v-for="child in visibleChildren" :key="child.id">
      <!-- 子菜单的子项 -->
      <SidebarItem
        v-if="child.children && child.children.length > 0 && child.type === 'C'"
        :menu="child"
        :base-path="resolvePath(child.routePath)"
      />

      <!-- 叶子菜单项 -->
      <el-menu-item v-else :index="resolvePath(child.routePath)">
        <el-icon>
          <component :is="getIconName(child.icon)" v-if="child.icon" />
        </el-icon>
        <template #title>{{ child.name }}</template>
      </el-menu-item>
    </template>
  </el-sub-menu>
</template>

<script setup>
import { computed } from 'vue';

const props = defineProps({
  menu: { type: Object, required: true },
  basePath: { type: String, default: '' },
});

// 可见子菜单
const visibleChildren = computed(() => {
  if (!props.menu.children) return [];
  return props.menu.children
    .filter((m) => m.visible !== 0 && m.type !== 'B')
    .sort((a, b) => (a.sort || 0) - (b.sort || 0));
});

// 是否只有一个子菜单
const isSingleChild = computed(() => {
  return visibleChildren.value.length === 1 && props.menu.type === 'C';
});

// 唯一子菜单
const singleChild = computed(() => {
  return visibleChildren.value[0] || props.menu;
});

// 唯一子菜单的完整路径
const singleChildFullPath = computed(() => {
  return resolvePath(singleChild.value.routePath);
});

// 解析完整路径
function resolvePath(routePath) {
  if (routePath.startsWith('/')) return routePath;
  return `${props.basePath}/${routePath}`.replace(/\/+/g, '/');
}

const iconMap = {
  user: 'UserFilled',

  article: 'Document',
  comment: 'ChatDotRound',
  file: 'FolderOpened',
  log: 'Tickets',
  monitor: 'Monitor',
};

function getIconName(icon) {
  return iconMap[icon] || icon || 'Menu';
}
```

## 7. keep-alive 缓存策略

```vue
<template>
  <router-view v-slot="{ Component, route }">
    <keep-alive :include="cachedViews">
      <component
        :is="Component"
        v-if="shouldKeepAlive(route)"
        :key="route.fullPath"
      />
    </keep-alive>
    <component
      :is="Component"
      v-if="!shouldKeepAlive(route)"
      :key="route.fullPath"
    />
  </router-view>
</template>

<script setup>
import { ref, computed, watch } from 'vue';
import { useRoute } from 'vue-router';

const route = useRoute();

// 缓存的组件名称列表（动态管理）
const cachedViews = ref([]);

/**
 * 判断当前路由是否需要缓存
 * 根据 keepAlive 字段决定
 */
function shouldKeepAlive(currentRoute) {
  return currentRoute.meta?.keepAlive === true;
}

/**
 * 监听路由变化，动态管理缓存列表
 */
watch(
  () => route.name,
  (newName) => {
    if (!newName) return;

    const needsCache = route.meta?.keepAlive === true;

    if (needsCache && !cachedViews.value.includes(newName)) {
      // 添加到缓存
      cachedViews.value.push(newName);
    } else if (!needsCache) {
      // 从缓存中移除
      cachedViews.value = cachedViews.value.filter(
        (name) => name !== newName
      );
    }
  },
  { immediate: true }
);

/**
 * 手动移除缓存（用于刷新当前页面）
 */
function removeCache(routeName) {
  cachedViews.value = cachedViews.value.filter(
    (name) => name !== routeName
  );
}

/**
 * 清空所有缓存
 */
function clearAllCache() {
  cachedViews.value = [];
}

defineExpose({ removeCache, clearAllCache });
</script>
```

## 8. 多标签页 (Tabs) 建议

后台管理系统推荐使用多标签页模式，支持同时打开多个页面并保持各自状态。

```vue
<template>
  <div class="tabs-view">
    <!-- 标签栏 -->
    <div class="tabs-bar">
      <div
        v-for="tag in visitedViews"
        :key="tag.fullPath"
        class="tab-item"
        :class="{ active: isActive(tag) }"
        @click="goto(tag)"
      >
        <span class="tab-title">{{ tag.title }}</span>
        <span
          class="tab-close"
          v-if="!isAffix(tag)"
          @click.stop="closeTag(tag)"

          &times;
        </span>

    </div>

    <!-- 页面内容（配合 keep-alive） -->
    <div class="tabs-content">
      <router-view v-slot="{ Component }">
        <keep-alive :include="cachedViewNames">
          <component :is="Component" :key="route.fullPath" />
        </keep-alive>
      </router-view>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, watch } from 'vue';
import { useRoute, useRouter } from 'vue-router';

const route = useRoute();
const router = useRouter();

// 已访问的视图列表
const visitedViews = ref([]);

// 固定标签（不可关闭）
const affixTags = [
  { path: '/dashboard', name: 'Dashboard', title: '首页', fullPath: '/dashboard' },
];

// 缓存组件名列表
const cachedViewNames = computed(() => {
  return visitedViews.value
    .filter((v) => v.keepAlive)
    .map((v) => v.name)
    .filter(Boolean);
});

// 初始化固定标签
visitedViews.value = [...affixTags];

// 判断是否激活
function isActive(tag) {
  return tag.fullPath === route.fullPath;
}

// 判断是否固定标签
function isAffix(tag) {
  return affixTags.some((a) => a.path === tag.path);
}

// 跳转到标签
function goto(tag) {
  router.push(tag.fullPath);
}

// 关闭标签
function closeTag(tag) {
  const index = visitedViews.value.findIndex(
    (v) => v.fullPath === tag.fullPath
  );
  if (index === -1) return;

  visitedViews.value.splice(index, 1);

  // 如果关闭的是当前激活的标签，跳转到相邻标签
  if (isActive(tag)) {
    const lastView = visitedViews.value[visitedViews.value.length - 1];
    if (lastView) {
      router.push(lastView.fullPath);
    } else {
      router.push('/');
    }
  }
}

// 关闭其他标签
function closeOtherTags(tag) {
  visitedViews.value = visitedViews.value.filter(
    (v) => isAffix(v) || v.fullPath === tag.fullPath
  );
}

// 关闭所有标签
function closeAllTags() {
  visitedViews.value = [...affixTags];
  router.push(affixTags[0].fullPath);
}

// 监听路由变化，添加新标签
watch(
  () => route.path,
  () => {
    if (route.meta?.hidden) return; // 隐藏路由不添加标签

    const exists = visitedViews.value.some(
      (v) => v.fullPath === route.fullPath
    );

    if (!exists) {
      visitedViews.value.push({
        path: route.path,
        fullPath: route.fullPath,
        name: route.name,
        title: route.meta?.title || route.name || '未知页面',
        keepAlive: route.meta?.keepAlive === true,
      });
    }
  },
  { immediate: true }
);
</script>
```

### 与 keepAlive 配合策略

| 操作 | 缓存行为 |
|------|----------|
| 打开新标签 | 如果 `keepAlive=1`，组件加入缓存 |
| 切换标签 | 缓存组件不销毁，直接从缓存恢复 |
| 关闭标签 | 从 `cachedViews` 中移除对应组件名，KeepAlive 自动销毁 |
| 刷新当前页 | 先移除缓存，再重新添加，触发组件重新挂载 |

前端处理结果：
- `C` 类型生成 Layout 包裹路由，侧边栏显示为分组标题
- `M` 类型动态 `import()` 组件，注册为 Layout 的子路由
- `B` 类型提取 `perm` 到权限集合，用于 `v-perm` 指令控制按钮显示
- `visible=0` 的文章详情页路由仍可访问，但不显示在侧边栏
