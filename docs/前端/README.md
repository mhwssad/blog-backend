## 快速开始

### 第 1 步：配置后端地址

在后端启动后，确认以下地址可访问：

- 后端 API：`http://localhost:8000`
- API 文档（Knife4j）：`http://localhost:8000/doc.html`
- OpenAPI JSON：`http://localhost:8000/v3/api-docs`

在前端项目中配置 API 基础路径，确保所有请求指向正确的后端地址。

### 第 2 步：实现登录

实现账号密码登录页，调用 `POST /api/auth/login` 获取 Token。登录成功后，将返回的 `accessToken` 和 `refreshToken` 持久化存储（推荐 localStorage），并在后续请求的 `Authorization` 头中携带 `Bearer <accessToken>`。

详见 [01-项目概览与认证.md](./01-项目概览与认证.md)。

### 第 3 步：获取菜单路由

登录成功后调用 `GET /api/auth/current-user-menus` 获取当前用户的菜单树，根据返回的 `AuthMenuInfo` 动态注册前端路由。菜单类型分三种：C（目录）、M（菜单页面）、B（按钮权限）。

详见 [08-动态菜单与权限路由.md](./08-动态菜单与权限路由.md)。

### 第 4 步：按页面开发

参照页面设计文档逐页实现：

- 前台页面：[02-前台页面设计.md](./02-前台页面设计.md)
- 后台页面：[03-后台页面设计.md](./03-后台页面设计.md)
- API 端点速查：[04-API端点速查表.md](./04-API端点速查表.md)

### 第 5 步：处理错误码

在全局请求拦截器中统一处理错误码，特别注意：

- `40101`（凭证错误）和 `40102`（未登录）触发 Token 刷新或跳转登录
- `40104`（账号锁定）展示锁定时长提示
- `42900`（请求限流）展示友好提示并延迟重试
- `40001`（参数校验失败）提取错误信息回填到表单

详见 [09-错误码参考.md](./09-错误码参考.md)。
