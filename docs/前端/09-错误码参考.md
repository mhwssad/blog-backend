# 错误码参考

> 后端所有错误码及前端处理策略。HTTP 响应体统一为 `Result<T>` 格式，`code` 为业务错误码，非 HTTP 状态码。

## HTTP 状态码与业务错误码映射

| HTTP Status | 业务 code 范围       | 含义        |
|-------------|------------------|-----------|
| 200         | 200              | 成功        |
| 400         | 40001-40011      | 参数校验/请求错误 |
| 401         | 40100-40115      | 认证错误      |
| 403         | 40300-40303      | 权限错误      |
| 404         | 40400-40401      | 资源不存在     |
| 429         | 42900            | 频率限制      |
| 500         | 500, 50000-50020 | 系统错误      |

## 通用错误码 (ResultErrorCode)

## 前端全局错误处理策略

### 错误类型与处理策略矩阵

| 错误类型     | code 范围            | 处理策略             |
|----------|--------------------|------------------|
| 成功       | 200                | 正常处理 data        |
| 参数错误     | 40001-40011        | 表单字段级提示          |
| Token 过期 | 40101              | 自动刷新 → 重试原请求     |
| 认证失败     | 40100, 40102-40115 | 清除 token → 跳转登录页 |
| 权限不足     | 40300-40303        | 显示 403 页面或提示     |
| 资源不存在    | 40400-40401        | 显示 404 页面        |
| 频率限制     | 42900, 40011       | Toast 提示稍后重试     |
| 文件错误     | 71001-71026        | Toast 提示具体错误     |
| 存储错误     | 70001-70007        | Toast 提示稍后重试     |
| 系统错误     | 500, 50000-50020   | Toast 提示系统繁忙     |

### Axios 拦截器完整实现

```javascript
import axios from 'axios'

const request = axios.create({
  baseURL: 'http://localhost:8000',
  timeout: 30000,
})

let isRefreshing = false
let pendingRequests = []

// 请求拦截器 — 注入 Token
request.interceptors.request.use(config => {
  const token = localStorage.getItem('accessToken')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器 — 统一错误处理
request.interceptors.response.use(
  response => {
    const { code, message, data } = response.data

    if (code === 200) {
      return data
    }

    // Token 过期 → 尝试刷新
    if (code === 40101) {
      return handleTokenRefresh(response.config)
    }

    // 其他认证错误 → 跳转登录
    if (code >= 40100 && code <= 40115) {
      clearAuthAndRedirect()
      return Promise.reject(new Error(message))
    }

    // 权限错误
    if (code >= 40300 && code <= 40303) {
      router.push('/403')
      return Promise.reject(new Error(message))
    }

    // 资源不存在
    if (code === 40400) {
      router.push('/404')
      return Promise.reject(new Error(message))
    }

    // 频率限制
    if (code === 42900 || code === 40011) {
      showToast('操作过于频繁，请稍后重试')
      return Promise.reject(new Error(message))
    }

    // 文件/存储模块错误
    if ((code >= 71001 && code <= 71026) || (code >= 70001 && code <= 70007)) {
      showToast(message || '文件操作失败')
      return Promise.reject(new Error(message))
    }

    // 参数校验错误
    if (code >= 40001 && code <= 40011) {
      showToast(message || '参数错误')
      return Promise.reject(new Error(message))
    }

    // 其他错误
    showToast(message || '操作失败')
    return Promise.reject(new Error(message))
  },
  error => {
    // HTTP 网络错误
    showToast('网络异常，请检查网络连接')
    return Promise.reject(error)
  }
)

// Token 刷新逻辑（防止并发刷新）
async function handleTokenRefresh(failedConfig) {
  if (!isRefreshing) {
    isRefreshing = true
    try {
      const refreshToken = localStorage.getItem('refreshToken')
      const { data } = await axios.post('/api/auth/refresh', { refreshToken })
      localStorage.setItem('accessToken', data.accessToken)
      localStorage.setItem('refreshToken', data.refreshToken)
      // 重试所有排队的请求
      pendingRequests.forEach(cb => cb(data.accessToken))
      pendingRequests = []
    } catch {
      clearAuthAndRedirect()
      return Promise.reject(new Error('刷新 Token 失败'))
    } finally {
      isRefreshing = false
    }
  }

  // 将当前请求加入等待队列
  return new Promise(resolve => {
    pendingRequests.push(token => {
      failedConfig.headers.Authorization = `Bearer ${token}`
      resolve(request(failedConfig))
    })
  })
}

function clearAuthAndRedirect() {
  localStorage.removeItem('accessToken')
  localStorage.removeItem('refreshToken')
  router.push('/login')
}

function showToast(msg) {
  // 使用项目 UI 框架的 Toast 组件
  console.warn('[Error]', msg)
}
```
