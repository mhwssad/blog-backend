# 数据模型与枚举值

> 前端硬编码需要的所有状态字段和枚举值。建议在前端创建对应的常量文件（如 `constants/enums.ts`）统一管理。

## 5.1 通用状态枚举

### 用户状态 (SysUser.status)

| 值 | 说明 |
|---|----|
| 0 | 禁用 |
| 1 | 启用 |

### 性别 (SysUser.gender)

| 值 | 说明 |
|---|----|
| 0 | 未知 |
| 1 | 男  |
| 2 | 女  |
| 3 | 保密 |

### 逻辑删除标记

| 值 | 说明  |
|---|-----|
| 0 | 未删除 |
| 1 | 已删除 |

### 角色状态 (SysRole.status)

| 值 | 说明 |
|---|----|
| 0 | 停用 |
| 1 | 正常 |

### 数据权限 (SysRole.dataScope)

| 值 | 说明     |
|---|--------|
| 1 | 所有数据   |
| 2 | 部门及子部门 |
| 3 | 本部门    |
| 4 | 本人     |
| 5 | 自定义    |

## 5.2 菜单枚举

### 菜单类型 (SysMenu.type)

| 值 | 说明 | 前端处理    |
|---|----|---------|
| C | 目录 | 渲染子菜单容器 |
| M | 菜单 | 对应页面组件  |
| B | 按钮 | 按钮级权限控制 |

### 菜单布尔字段

| 字段         | 0     | 1    |
|------------|-------|------|
| alwaysShow | 不始终显示 | 始终显示 |
| keepAlive  | 不缓存   | 开启缓存 |
| visible    | 隐藏    | 显示   |
