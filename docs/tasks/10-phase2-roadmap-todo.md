# 第二期任务规划清单

本文档用于承接第一阶段未完成项和已明确后置能力，作为第二期开发入口。第一期已完成博客、聊天、AI 基础问答、举报治理、通知偏好、后台看板、数据库结构对齐和结构治理等主线能力，第二期默认在这些能力稳定的基础上推进。

## 1. 当前状态

**当前阶段：待开始。**

```
第一期基线:
  ✅ 用户注册登录、RBAC、作者申请、等级经验、超级管理员安全
  ✅ 博客内容生命周期、审核、可见范围、系列文章、附件引用
  ✅ 大厅频道、主题频道、群聊增强、频道与帖子轻量挂接
  ✅ AI 基础问答、渠道配置、额度控制、调用统计、数据范围控制
  ✅ 举报处理、会话级禁言、账号封禁、审计留档
  ✅ 通知偏好、通知投递过滤、后台首批数据看板
  ✅ 数据库结构和源码结构已完成当前阶段对齐

第二期重点:
  ⏳ 论坛正式模块
  ⏳ 知识库 / RAG
  ⏳ agents
  ⏳ 外部博客迁移方案与导入工具
  ⏳ 治理、通知、性能和测试补强
```

## 2. P0：第二期主线候选

### 2.1 论坛正式模块

- [ ] 设计论坛版块、帖子、回复、帖子状态、置顶 / 精华等基础结构。
- [ ] 落地帖子创建、编辑、删除、详情、分页和公开查询接口。
- [ ] 接入频道挂接能力，复用第一期 `forum_post_channel_link` 的单频道挂接约束。
- [ ] 补齐帖子与频道双向跳转数据。
- [ ] 明确论坛与博客分类 / 标签是否继续解耦。
- [ ] 补齐服务级测试和接口文档。

### 2.2 知识库 / RAG

- [ ] 设计文章、论坛帖子、公开资料的知识源接入范围。
- [ ] 实现知识条目入库、更新、删除和失效策略。
- [ ] 接入向量化和索引构建流程。
- [ ] 实现检索增强上下文拼接。
- [ ] AI 回答返回引用来源。
- [ ] 管理端提供知识源开关和同步状态查询。
- [ ] 默认仍禁止读取用户私聊内容。

### 2.3 Agents

- [ ] 设计 agent 定义、工具配置、执行状态和任务日志模型。
- [ ] 支持管理端创建、启用、停用 agent。
- [ ] 支持用户发起 agent 任务。
- [ ] 接入异步任务状态流转。
- [ ] 任务完成后接入通知中心。
- [ ] 保持 agent 工具权限、数据读取范围和审计边界可控。

### 2.4 外部博客迁移

- [ ] 梳理外部博客字段到当前文章、分类、标签、附件、作者的映射。
- [ ] 设计导入校验、幂等和失败回滚策略。
- [ ] 实现导入工具或后台导入接口。
- [ ] 支持导入结果统计和错误明细下载。
- [ ] 补齐迁移前后的数据一致性验证。

## 3. P1：第一期遗留增强

### 3.1 治理与通知增强

- [ ] 扩展禁言范围：大厅、主题频道、群、全站。
- [ ] 举报处理结果通知举报人。
- [ ] 被举报人通知策略评估与实现。
- [ ] 通知接入时遵守用户通知设置。
- [ ] AI 任务完成通知投递前检查用户开关。

### 3.2 性能与体验补强

- [ ] 文件公开访问改为流式响应，避免一次性 `readAllBytes()`。
- [ ] 公开评论根评论查询改为分页返回。
- [ ] 文章关联过滤下沉到数据库查询，避免加载全量 ID。
- [ ] 关注数 / 粉丝数缓存评估与实现。
- [ ] WebSocket 本地推送并发发送保护。
- [ ] 后台看板高频指标评估聚合表或缓存。

### 3.3 账号自服务接口

- [ ] 用户资料更新接口。
- [ ] 用户修改密码接口。
- [ ] 找回密码 / 重置密码接口。
- [ ] 用户搜索接口，支撑前端全局搜索的用户 Tab。
- [ ] 友情链接后台配置与公开查询接口评估。

## 4. P2：测试、结构与清爽性

- [ ] 新增表对应 Repository 基础测试或服务级测试。
- [ ] 对新增字段涉及的查询补服务级回归。
- [ ] 对权限菜单初始化脚本做人工核对或脚本级校验。
- [ ] 补强 auth、article、content、chat、file 的关键异常路径测试。
- [ ] 为核心 Controller 补充权限测试，验证 `@PreAuthorize` 注解。
- [ ] 减少测试中的 `lenient()` stub，关键测试增加状态断言。
- [ ] 按需处理 P3 代码质量项，不与二期主线混提交。

## 5. 推荐执行顺序

1. 先选定一个主线：论坛、RAG、agents、迁移工具四选一优先推进。
2. 主线推进期间同步处理直接依赖的遗留增强，不做无关重构。
3. 每完成一个独立能力，更新对应 API 文档、任务清单和需求文档。
4. 性能、测试和清爽性任务按影响范围拆小提交，避免和主功能混在一起。

## 6. 完成标志

- 至少一个第二期主线能力完成业务闭环、接口文档和服务级测试。
- 第一阶段遗留增强已按优先级完成或明确继续后置。
- 新增能力不破坏第一期博客、聊天、AI、举报、通知和后台看板基线。
