## 3. T13 通知设置中心

### 3.1 目标

- 用户可按通知类型开启或关闭提醒。
- 通知投递前按用户偏好过滤。
- 新用户自动生成默认通知设置。

### 3.2 通知类型

- [x] `comment_me`：评论我。
- [x] `like_me`：点赞我。
- [x] `collect_article`：收藏我文章。
- [x] `follow_me`：有人关注我。
- [x] `private_message`：收到私聊。
- [x] `group_mention`：群聊有人 @ 我。
- [x] `channel_announcement`：频道公告。
- [x] `system_announcement`：系统公告。
- [x] `ai_task_done`：AI 任务完成。

### 3.3 数据与模型

- [x] 复核 `sys_user_notification_setting` 是否覆盖用户 ID、通知类型、是否启用、更新时间。
- [x] 定义新用户默认通知偏好。
- [x] 对缺失配置采用默认开启或按类型默认策略。
- [x] 投递服务只关心通知类型和接收用户，不关心业务来源细节。

### 3.4 用户侧接口

- [x] `GET /api/user/notification-settings`：查询我的通知设置。
- [x] `PUT /api/user/notification-settings`：批量更新通知设置。
- [x] `PUT /api/user/notification-settings/{type}`：单独更新某类通知。
- [x] 非法通知类型被拦截。

### 3.5 投递接入

- [x] 评论我通知投递前检查开关。
- [x] 点赞我通知投递前检查开关。
- [x] 收藏我文章通知投递前检查开关。
- [x] 关注我通知投递前检查开关。
- [x] 私聊通知投递前检查开关。
- [x] 群聊 @ 通知投递前检查开关。
- [x] 频道公告通知投递前检查开关。
- [x] 系统公告通知按策略处理是否允许关闭。
- [ ] AI 任务完成通知投递前检查开关。

### 3.6 测试

- [ ] 新用户默认通知设置生成。
- [ ] 用户关闭关注通知后，新粉丝通知不投递。
- [ ] 用户关闭私聊通知后，私聊通知不投递。
- [ ] 非法通知类型更新被拦截。
- [ ] 缺失设置时按默认策略投递。

## 4. T14 数据统计看板首批指标

### 4.1 目标

- 后台提供平台核心数据概览。
- 指标口径清晰可追溯。
- 第一阶段优先做日 / 周 / 总量统计。

### 4.2 首批指标

- [x] 注册用户数。
- [x] 活跃用户数。
- [x] 作者数量。
- [x] 发文数。
- [x] 待审核文章数。
- [x] 评论数。
- [x] 点赞数。
- [x] 收藏数。
- [x] 聊天消息数。
- [x] 大厅消息数。
- [x] 群聊数量。
- [x] AI 调用数。
- [x] 举报单数。
- [x] 待处理举报数。

### 4.3 后台接口

- [x] `GET /api/sys/dashboard/overview`：核心概览。
- [x] `GET /api/sys/dashboard/content`：内容统计。
- [x] `GET /api/sys/dashboard/community`：社区统计。
- [x] `GET /api/sys/dashboard/ai`：AI 调用统计。
- [x] `GET /api/sys/dashboard/governance`：治理统计。
- [x] 支持时间范围：今日、本周、本月、自定义、全部。

### 4.4 指标口径

- [x] 活跃用户定义：登录、发文、评论、聊天或 AI 调用任一行为。
- [x] 发文数按创建时间统计。
- [x] 聊天消息数包含已撤回消息。
- [x] AI 调用数按请求次数统计，并区分成功 / 失败。
- [x] 举报单数按提交时间统计。

### 4.5 性能策略

- [x] 第一阶段可先实时查询。
- [x] 高频指标预留后续聚合表或缓存。
- [x] 大时间范围查询需要限制最大跨度。
- [x] 后台看板接口需要权限控制。

### 4.6 测试

- [ ] 管理员可查看看板。
- [ ] 无权限用户访问看板被拦截。
- [ ] 今日 / 本周 / 自定义时间范围统计正确。
- [ ] 空数据时返回 0 而不是异常。
- [ ] AI 和举报指标按状态正确聚合。
