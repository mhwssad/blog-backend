## 2. P0 知识源边界

- [x] 明确可入库知识源：公开文章、作者公开资料、论坛帖子、后台维护知识条目。
- [x] 明确不可入库内容：用户私聊、私密文章、白名单文章、被删除 / 隐藏内容。
- [x] 设计知识源类型枚举。
- [x] 设计知识条目表：来源类型、来源 ID、标题、摘要、内容快照、状态、版本。
- [x] 设计知识同步任务表：任务类型、状态、错误信息、重试次数。
- [x] 后台配置每类知识源是否启用。

## 6. 权限与安全

- [x] agent 数据读取范围必须复用 AI 数据范围配置。
- [ ] 高风险 agent 配置变更写审计日志。
- [x] 工具授权范围必须按 agent / 场景 / 用户权限 / 数据范围收口，不允许默认全局可用。
- [x] MCP 服务配置、鉴权信息和工具授权调整必须写审计日志。
- [ ] RAG 不得返回用户无权访问的内容引用。
- [ ] 私聊数据读取继续默认关闭。
- [ ] 额度消耗规则覆盖普通 AI 问答、RAG 问答和 agent 任务。

## 9. 完成标志

- AI 能基于公开知识源回答并返回引用。
- 后台可控制知识源和 RAG 开关。

- 数据权限、额度和审计边界生效。
