## 3. T01 作者申请与审核闭环

### 3.1 目标

- 注册用户可提交作者申请。
- 管理员可审核通过、拒绝或要求补充。
- 审核通过后用户获得作者增强权限。
- 审核记录可追溯。

### 3.2 数据与模型

- [x] 复核 `sys_author_application` 字段是否覆盖申请说明、擅长方向、个人简介、示例链接、审核状态、审核意见、审核人、审核时间。
- [x] 定义申请状态枚举：`pending`、`approved`、`rejected`、`need_more_info`。
- [x] 定义作者权限判断入口，避免业务服务散落判断 `roleCode` 或用户字段。
- [x] 明确审核通过后是授予作者角色、更新用户扩展字段，还是两者都做。

### 3.3 用户侧接口

- [x] `POST /api/user/author-applications`：提交作者申请。
- [x] `GET /api/user/author-applications/latest`：查看最近一次申请状态。
- [x] `GET /api/user/author-applications`：查看个人申请记录。
- [x] 参数校验：申请说明、内容方向、简介长度、示例链接格式。
- [x] 重复申请规则：存在 `pending` / `need_more_info` 时禁止重复新建，允许补充资料。

### 3.4 后台接口

- [x] `GET /api/sys/author-applications`：分页筛选申请记录。
- [x] `GET /api/sys/author-applications/{id}`：查看申请详情。
- [x] `PUT /api/sys/author-applications/{id}/review`：审核通过、拒绝、要求补充。
- [x] 审核通过后授予作者权限，并记录审核人、审核时间、审核意见。
- [x] 已终态申请禁止普通管理员重复审核。

### 3.5 权限与联动

- [x] 作者权限接入文章系列 / 专栏能力。
- [x] 作者权限接入更高发文配额或后续配额预留。
- [x] 作者主页增强字段先预留查询入口，前端展示可后续补齐。
- [x] 超级管理员可修正异常作者状态。

### 3.6 测试

- [x] 用户提交作者申请成功。
- [x] 有待审核申请时重复提交被拦截。
- [x] 管理员审核通过后作者权限生效。
- [x] 管理员审核拒绝后用户仍为普通用户。
- [x] 非授权管理员审核被拦截。
- [x] 已审核申请重复审核被拦截。

当前阶段说明：
测试已回补完成，服务级测试与权限测试已覆盖。

## 4. T02 用户等级与经验体系

### 4.1 目标

- 建立 1 到 10 级用户等级体系。
- 支持登录、发文章、评论、点赞 / 被点赞、聊天活跃带来经验。
- 支持每日总经验上限、聊天经验上限、评论经验上限。
- 等级影响大厅发言、主题频道发言、建群、频道申请和 AI 次数。

### 4.2 数据与模型

- [x] 复核 `sys_user.user_level`、`experience_points`、`level_updated_at` 是否满足当前展示与计算。
- [x] 新增或复用经验流水结构，记录来源类型、来源业务 ID、经验值、入账日期。
- [x] 定义经验来源枚举：`daily_login`、`article_publish`、`comment_create`、`like_given`、`like_received`、`chat_message`。
- [x] 定义等级配置模型：等级、经验阈值、称号、权限门槛、AI 日额度。
- [x] 定义每日上限配置模型：总上限、聊天上限、评论上限、单来源开关。

### 4.3 服务入口

- [x] 新增 `UserExperienceService` 作为统一经验入账入口。
- [x] 入账接口支持幂等键，避免重复发放经验。
- [x] 等级计算由统一服务处理，不在业务模块内自行计算。
- [x] 等级变更时更新 `sys_user.user_level`、`experience_points`、`level_updated_at`。

### 4.4 行为接入

- [x] 登录成功后触发每日登录经验。
- [x] 文章发布成功后触发发文经验。
- [x] 评论发布成功后触发评论经验。
- [x] 点赞成功后触发主动点赞经验。
- [x] 内容被点赞时触发被点赞经验。
- [x] 聊天有效发言成功后触发聊天经验。

### 4.5 权限联动

- [x] 大厅频道发言前校验等级门槛。
- [x] 主题频道发言前校验等级门槛。
- [x] 创建群聊前校验等级门槛。
- [x] 申请创建频道前校验等级门槛。
- [x] AI 每日额度按用户等级计算。

### 4.6 后台管理

- [x] 管理端查看用户等级和经验。
- [x] 管理端查看用户经验来源汇总。
- [x] 管理端手动调整等级或经验，并写入审计日志。
- [x] 管理端配置经验来源开关和每日上限。

### 4.7 测试

- [x] 同一用户每日登录经验只发放一次。
- [x] 聊天经验达到单项上限后不再增加。
- [x] 总经验达到每日上限后所有来源停止入账。
- [x] 幂等键重复提交不重复发经验。
- [x] 等级提升后权限门槛立即生效。
- [x] 第一阶段不自动降级。

## 5. T12 超级管理员安全与审计增强

### 5.1 目标

- 明确唯一超级管理员账号规则。
- 超级管理员必须启用双重验证。
- 高风险动作必须审计。
- 支持超级管理员接管账号和修改其他管理员权限。

### 5.2 账号与权限

- [x] 明确超级管理员识别方式：`code='admin'` 角色 + `SuperAdminVerifier`。
- [x] 确保超级管理员唯一性校验：通过角色编码集合检查。
- [x] 超级管理员必须绑定必要邮箱：`2FA` 发送前校验邮箱非空。
- [x] 普通管理员只能在授权范围内操作：`@PreAuthorize` + 服务层 `SuperAdminVerifier` 双重校验。
- [x] 普通管理员不能绕过状态和权限限制。

### 5.3 双重验证

- [x] 定义高风险动作二次验证入口：`POST /api/admin/2fa/send-code` + `/2fa/verify`。
- [x] 支持邮箱验证码的第一阶段实现方式：`TwoFactorService`。
- [x] 二次验证通过后设置短期操作票据：UUID ticket 存 Redis 30min。
- [x] 操作票据过期后必须重新验证：`validateTicket` 检查 Redis key 存在性。

### 5.4 高风险动作

- [x] 修改其他管理员权限：`assignRolesWithAudit`。
- [x] 接管任意账号：`AccountTakeoverService`。
- [x] 封禁 / 解封用户：`banUser` / `unbanUser` + Token 黑名单。
- [x] 手动调整用户等级或经验：`adjustLevel` / `adjustExperience`。
- [x] 终结举报流程：待举报模块业务落地后接入。
- [x] 修改 AI 高风险配置：待 AI 配置模块业务落地后接入。

### 5.5 审计日志

- [x] 审计记录操作人、目标对象、操作类型、前后状态、IP、User-Agent、二次验证结果：`sys_audit_log` 表 + `SysAuditLogService`。
- [x] 审计日志不可由普通管理员删除：接口层无 DELETE 端点 + `SuperAdminVerifier.requireSuperAdmin()`。
- [x] 超级管理员审计查询需要独立权限：`sys:audit:query` 权限 + 超管校验。

### 5.6 测试

- [x] 非超级管理员接管账号被拦截。
- [x] 超级管理员缺少二次验证时高风险动作被拦截。
- [x] 二次验证通过后可执行高风险动作。
- [x] 高风险动作缺少审计写入时业务失败或测试报警。
