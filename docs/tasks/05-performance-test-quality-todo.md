# 性能、测试与代码质量补强任务清单

本文档用于规划第二期横向质量任务。此类任务不作为单独产品主线，但应按风险穿插执行，避免影响第一期基线和第二期新增主线。

## 1. 当前状态

**当前阶段：待开始。**

```
可复用基线:
  ✅ 编译通过
  ✅ 主要业务模块已有服务级测试基础
  ✅ 第一至第三阶段代码审查整改已完成

待建设:
  ⏳ 性能热点处理
  ⏳ 权限和异常路径测试补强
  ⏳ 代码质量清爽性改进
```

## 2. P0 性能风险

- [ ] 文件公开访问改为流式响应，避免一次性读取全部字节。
- [ ] 公开评论根评论查询改为分页。
- [ ] 文章关联过滤下沉到数据库查询，避免加载全量 ID。
- [ ] WebSocket 本地推送增加并发发送保护。
- [ ] 后台大范围统计限制最大跨度或使用聚合策略。

## 3. P1 缓存与索引

- [ ] 评估关注数 / 粉丝数缓存。
- [ ] 关注 / 取关时失效缓存。
- [ ] Redis Key 使用统一工具构建并设置 TTL。
- [ ] `sys_menu` 增加权限标识索引。
- [ ] `blog_article` 核心查询索引覆盖审核状态和可见范围。
- [ ] `sys_log` 常用查询索引优化。

## 4. P1 测试补强

- [ ] 账号加载、账号不存在、账号禁用测试。
- [ ] 2FA 验证流程测试。
- [ ] 账号接管逻辑测试。
- [ ] 作者权限校验测试。
- [ ] 文章 CRUD 和状态机异常路径测试。
- [ ] 聊天消息发送异常路径测试。
- [ ] 频道加入 / 退出测试。
- [ ] 文件上传分片 / 秒传测试。
- [ ] article、content、auth 核心 Controller 权限测试。

## 5. P2 代码质量

- [ ] 无 TTL Redis 写入方法增加警告或废弃标记。
- [ ] 常量类统一目标类型常量。
- [ ] `ArticleStatusMachine` 魔法数字改为枚举或显式常量。
- [ ] `SecurityExceptionHandler` 重复异常处理收口。
- [ ] `RedisKeyUtils.build()` 空结果改为抛异常。
- [ ] 异常处理器中环境名硬编码改为常量。
- [ ] 按需拆分仍然偏重的支撑类。

## 6. 执行规则

- 性能和测试任务应按模块小提交，不与论坛、RAG、agents 等主功能混提交。
- 涉及接口返回结构变化时必须同步 API 文档。
- 涉及数据库索引时，当前开发阶段默认直接更新原始建表脚本。
- 每个质量任务完成后更新本文档状态。

## 7. 完成标志

- 已处理影响稳定性或资源占用的 P0 性能风险。
- 核心权限和异常路径测试覆盖明显增强。
- 代码质量改动不改变业务语义，且编译通过。
