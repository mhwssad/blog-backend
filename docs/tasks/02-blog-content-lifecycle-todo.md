
## 3. T03 文章状态、可见范围与定时发布

### 3.1 目标

- 文章不再只按简单发布状态判断展示。
- 状态和可见范围分开表达。
- 定时发布和审核中内容不会误进公开列表。

### 3.2 状态模型

- [ ] 复核 `blog_article.review_status` 是否覆盖 `draft`、`pending`、`approved`、`rejected`。
- [ ] 复核 `blog_article.visibility_scope` 是否覆盖 `public`、`private`、`whitelist`、`login_required`。
- [ ] 复核 `blog_article.scheduled_publish_time` 是否支持定时发布。
- [ ] 明确原有 `status` 字段与新增审核 / 可见字段的关系。
- [ ] 定义统一文章状态机，避免保存、审核、查询各自判断。

### 3.3 用户侧能力

- [ ] 创建文章时可保存草稿。
- [ ] 编辑文章时可更新可见范围。
- [ ] 用户可提交文章审核。
- [ ] 用户可查看自己的草稿、审核中和已拒绝文章。
- [ ] 用户可设置定时发布时间。
- [ ] 用户可配置白名单用户。

### 3.4 公开查询联动

- [ ] 公开列表只展示已发布且满足公开可见规则的文章。
- [ ] 文章详情按访客、登录用户、作者、管理员分别校验访问权限。
- [ ] 白名单文章只允许作者、白名单用户和管理员查看。
- [ ] 仅自己可见文章只允许作者和具备权限的管理员查看。
- [ ] 登录可见文章对访客不可见。

### 3.5 定时发布

- [ ] 新增或复用定时任务扫描到期文章。
- [ ] 到期后将文章切换到可公开展示状态。
- [ ] 定时发布任务应幂等，重复执行不造成异常状态。
- [ ] 定时发布失败需要日志记录。

### 3.6 评论与互动联动

- [ ] 不可见文章禁止评论、点赞、收藏。
- [ ] 访客只可查看公开文章下一级评论。
- [ ] 登录用户可展开完整评论树。
- [ ] 草稿、审核中、已拒绝文章的评论入口关闭。

### 3.7 测试

- [ ] 草稿不会进入公开列表。
- [ ] 审核中文章不会进入公开列表。
- [ ] 已拒绝文章不会进入公开列表。
- [ ] 白名单用户可访问白名单文章，非白名单用户被拦截。
- [ ] 定时文章未到时间不可公开，到时间后可公开。
- [ ] 访客只返回一级评论，登录用户返回完整评论树。

## 4. T04 文章审核流转

### 4.1 目标

- 建立送审、审核通过、审核拒绝、重提审核的完整流程。
- 审核动作留痕。
- 审核状态与文章展示一致。

### 4.2 数据与模型

- [ ] 复核 `blog_article_review_log` 字段是否覆盖文章 ID、审核动作、审核人、审核意见、审核前状态、审核后状态。
- [ ] 定义审核动作枚举：`submit`、`approve`、`reject`、`resubmit`。
- [ ] 审核日志只追加不覆盖。

### 4.3 用户侧流程

- [ ] 用户提交审核。
- [ ] 拒绝后允许编辑并重新提交。
- [ ] 审核中限制部分字段修改，或修改后自动回到草稿 / 待审核。
- [ ] 用户可查看审核意见。

### 4.4 后台流程

- [ ] 管理员查看待审核文章列表。
- [ ] 管理员查看审核详情和历史审核记录。
- [ ] 管理员审核通过。
- [ ] 管理员审核拒绝并填写原因。
- [ ] 超级管理员可修正异常审核状态。

### 4.5 测试

- [ ] 草稿可提交审核。
- [ ] 审核中重复提交被拦截。
- [ ] 审核通过后文章可按可见范围展示。
- [ ] 审核拒绝后文章不可公开展示。
- [ ] 拒绝后重新编辑并送审成功。
- [ ] 审核日志记录完整。

## 5. T05 专栏 / 系列文章能力

### 5.1 目标

- 作者可用系列组织长期内容。
- 文章详情可展示所属系列。
- 系列内文章顺序可控。

### 5.2 数据与模型

- [ ] 复核 `blog_article_series` 是否覆盖作者 ID、标题、简介、封面、状态、排序。
- [ ] 复核 `blog_article_series_item` 是否覆盖系列 ID、文章 ID、排序、加入时间。
- [ ] 系列归属必须校验作者权限。
- [ ] 同一文章在同一系列内不可重复加入。

### 5.3 用户侧接口

- [ ] `POST /api/user/article-series`：创建系列。
- [ ] `PUT /api/user/article-series/{id}`：编辑系列。
- [ ] `DELETE /api/user/article-series/{id}`：删除或归档系列。
- [ ] `POST /api/user/article-series/{id}/articles`：加入文章。
- [ ] `DELETE /api/user/article-series/{id}/articles/{articleId}`：移出文章。
- [ ] `PUT /api/user/article-series/{id}/articles/sort`：调整系列内排序。

### 5.4 公开接口

- [ ] 查询作者系列列表。
- [ ] 查询系列详情。
- [ ] 查询系列文章列表。
- [ ] 文章详情返回所属系列摘要。

### 5.5 权限规则

- [ ] 普通用户是否可创建系列按作者权限规则控制。
- [ ] 作者只能管理自己的系列。
- [ ] 系列内不可加入无权限访问或不属于自己的文章。
- [ ] 删除系列不删除文章，只解除关联。

### 5.6 测试

- [ ] 作者创建系列成功。
- [ ] 普通用户无作者权限时创建系列被拦截。
- [ ] 重复加入同一文章被拦截。
- [ ] 越权修改他人系列被拦截。
- [ ] 系列排序结果正确。
- [ ] 删除系列后文章仍存在。

## 6. 置顶与推荐收口

- [ ] 明确置顶和推荐是后台能力还是作者也可申请。
- [ ] 明确置顶 / 推荐字段和排序规则。
- [ ] 公开列表支持按置顶、推荐、热度、最新组合排序。
- [ ] 后台操作置顶 / 推荐必须审计。
- [ ] 补齐置顶过期或取消推荐后的展示回归测试。

## 7. 附件引用收口

- [ ] 文章保存时绑定附件引用。
- [ ] 文章更新时清理移除的附件引用。
- [ ] 文章删除时解除附件引用并触发文件生命周期收口。
- [ ] 白名单或私密文章附件访问不应绕过文章权限。
- [ ] 补齐附件越权访问和删除补偿测试。

## 8. 完成标志

- 文章状态、审核状态和可见范围边界清晰。
- 访客、登录用户、作者、管理员的文章与评论权限生效。
- 定时发布可稳定执行且不误公开。
- 专栏 / 系列文章可创建、排序、公开查询。
- 对应 API 文档、需求文档、进度文档已同步更新。
