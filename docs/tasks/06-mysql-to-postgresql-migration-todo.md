# MySQL 迁移 PostgreSQL 任务清单

本文档用于规划当前仓库从 MySQL 迁移到 PostgreSQL 的执行路径。迁移目标是先做到 PostgreSQL 可启动、可跑核心链路、可承接后续 RAG / pgvector，再决定是否移除 MySQL 作为默认数据库。

## 1. 当前状态

**当前阶段：P0 基础接入中。**

```
可复用基线:
  ✅ MySQL 原始建表脚本已按业务域拆分
  ✅ MyBatis-Plus Repository / Mapper 已形成主要数据访问边界
  ✅ RAG 基础依赖已接入，后续可使用 PostgreSQL + pgvector 承接向量检索

待建设:
  ✅ PostgreSQL 运行配置已新增 `pg` profile
  ⏳ PostgreSQL 原始建表脚本
  ⏳ Mapper SQL 方言兼容
  ⏳ 数据迁移和回归验证流程
```

## 2. 迁移原则

- 第一阶段保留 MySQL 配置和脚本，不直接删除现有能力。
- 新增 `pg` profile，通过 `spring.profiles.active=pg` 切换 PostgreSQL。
- PostgreSQL 脚本放在 `src/main/resources/postgresql`，不要覆盖 `src/main/resources/mysql`。
- 先保证空库建表和启动成功，再做历史数据迁移。
- 与业务功能开发分开提交，不混入论坛、RAG、治理等业务改动。
- 所有 Mapper SQL 方言调整必须有核心链路验证或测试覆盖。

## 5. P0 SQL 方言改造

重点扫描：

```powershell
rg "AUTO_INCREMENT|ON DUPLICATE|INSERT IGNORE|REPLACE INTO|IFNULL|DATE_FORMAT|JSON_|LAST_INSERT_ID|`" src/main/resources src/main/java
```

- [ ] `AUTO_INCREMENT` 改为 `GENERATED BY DEFAULT AS IDENTITY`。
- [ ] `datetime` 改为 `timestamp`。
- [ ] `tinyint` 按语义改为 `smallint` 或 `boolean`。
- [ ] `longtext` 改为 `text`。
- [ ] `json` 改为 `jsonb`。
- [ ] 去掉 MySQL 反引号。
- [ ] 删除 `ENGINE=InnoDB`、`DEFAULT CHARSET` 等 MySQL 表选项。
- [ ] `IFNULL` 改为 `COALESCE`。
- [ ] `ON DUPLICATE KEY UPDATE` 改为 `ON CONFLICT (...) DO UPDATE`。
- [ ] `INSERT IGNORE` 改为 `ON CONFLICT DO NOTHING`。
- [ ] `DATE_FORMAT` 改为 `to_char` 或 `date_trunc`。
