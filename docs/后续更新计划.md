# 后续更新计划

本文档用于指导当前仓库下一阶段的迭代方向。计划口径以模块级推进为主，面向项目内部协作与排期使用。

## 1. 计划基线

- 计划时间：起始于 2026-03-24 之后的下一阶段迭代
- 计划目标：在不盲目扩张模块面的前提下，优先补齐质量、文档、一致性和高风险链路

计划原则：`先保证已有模块可验证、可维护、可联调，再继续扩功能`、`先处理高风险链路，再处理低优先级体验增强`、
`接口行为变化必须同步更新前端文档`、`复杂实体构建优先收口到 mapstruct，其次使用 BeanConverterUtils`
